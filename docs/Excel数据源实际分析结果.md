## 二、P0级关键缺陷分析

### 缺陷1：学生数据分散在两个文件，无法关联

**问题描述：**
- 文件1有：学院名称、班级 ✓
- 文件2有：学号、班号 ✓
- 但两个文件**无法通过共同字段关联**

**尝试关联方案：**
1. **按姓名关联** → ❌ 不可靠（重名、缺失）
2. **按班级关联** → ❌ 格式不一致（文件1："电信(专升本)202403"，文件2："环境202201"）
3. **按专业关联** → ❌ 不精确（一对多关系）

**影响：**
- 无法生成完整的学生表（同时包含学号+学院+班级）
- 只能二选一：
  - 选文件1 → 有学院，但无学号（无法唯一标识）
  - 选文件2 → 有学号，但无学院（审批路由失效）

---

### 缺陷2：辅导员文件缺少负责班级清单

**问题描述：**
文件4只有：
```
学院, 职工号, 姓名, 电话
马克思主义学院, 20250015, 胡晓炀, 17386282440
```

**缺失信息：**
- 该辅导员负责哪些班级？
- 班级与辅导员的对应关系？

**影响：**
- 无法生成ClassMapping表
- 无法实现"辅导员只能审批自己负责班级的申请"
- 审批路由无法工作

---

### 缺陷3：部分宿管员无工号

**问题描述：**
文件3存在"暂未申请"职工号

**影响：**
- 这些宿管员无法导入系统（user_id必填）
- 需要人工补充或排除

### 方案B：使用单一文件+手工标记（折中）
