## 检查结果

### 1. SSH连接 ❌ 未通过

**问题**: Permission denied (publickey,password)

**原因**: 
- 199未配置SSH免密登录到196
- 或196未添加199的公钥

**解决方案**:
```bash
# 在199上执行

# 1. 生成SSH密钥（如果不存在）
[ -f ~/.ssh/id_ed25519 ] || ssh-keygen -t ed25519 -N "" -f ~/.ssh/id_ed25519 -C "dev-to-prod-199-196"

# 2. 复制公钥到196（需要196的root密码）
ssh-copy-id root@172.17.12.196

# 或手动操作：
# 在199上：cat ~/.ssh/id_ed25519.pub
# 在196上：mkdir -p ~/.ssh && echo "<公钥内容>" >> ~/.ssh/authorized_keys
# 在196上：chmod 700 ~/.ssh && chmod 600 ~/.ssh/authorized_keys
```

**验证**:
```bash
ssh root@172.17.12.196 "echo 'SSH连接成功'"
```

## 自动化检查脚本

将以下脚本保存为 `scripts/check-196-readiness.sh`:

```bash
#!/bin/bash
set -e

TARGET="root@172.17.12.196"
CHECKS_PASSED=0
CHECKS_FAILED=0

check() {
    echo -n "检查: $1 ... "
    if eval "$2" > /dev/null 2>&1; then
        echo "✅"
        ((CHECKS_PASSED++))
        return 0
    else
        echo "❌"
        ((CHECKS_FAILED++))
        return 1
    fi
}

echo "=== 196机器部署就绪检查 ==="
echo ""

# 1. SSH连接
check "SSH连接" "ssh -o ConnectTimeout=5 $TARGET 'exit'"

if [ $CHECKS_FAILED -eq 0 ]; then
    # 2. Docker安装
    check "Docker已安装" "ssh $TARGET 'command -v docker'"
    
    # 3. Docker运行
    check "Docker服务运行" "ssh $TARGET 'systemctl is-active docker'"
    
    # 4. 磁盘空间 (>20GB)
    check "磁盘空间充足" "ssh $TARGET '[ \$(df / | awk \"NR==2 {print \\\$4}\") -gt 20000000 ]'"
    
    # 5. 端口7787可用
    check "端口7787可用" "! ssh $TARGET 'lsof -i :7787' 2>/dev/null"
    
    # 6. 端口7788可用
    check "端口7788可用" "! ssh $TARGET 'lsof -i :7788' 2>/dev/null"
    
    # 7. rsync已安装
    check "rsync已安装" "ssh $TARGET 'command -v rsync'"
    
    # 8. curl已安装
    check "curl已安装" "ssh $TARGET 'command -v curl'"
fi

echo ""
echo "=== 检查结果 ==="
echo "通过: $CHECKS_PASSED"
echo "失败: $CHECKS_FAILED"
echo ""

if [ $CHECKS_FAILED -eq 0 ]; then
    echo "🎉 196机器已就绪，可以开始部署"
    exit 0
else
    echo "⚠️ 196机器未就绪，请解决上述问题"
    exit 1
fi
```

**使用方法**:
```bash
chmod +x scripts/check-196-readiness.sh
./scripts/check-196-readiness.sh
```
