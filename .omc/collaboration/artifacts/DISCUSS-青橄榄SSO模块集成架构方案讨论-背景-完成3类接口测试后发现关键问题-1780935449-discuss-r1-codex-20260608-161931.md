### 2.1 Credentials映射规则

### 2.2 接口用途分类

**类型1: 批量数据同步接口**
- 接口: 信息中心 `/api/open-api/user-center/tenant/auth-user-info`
- 特点: 一次返回大量数据(32060条), 分页查询
- 用途: 定期同步全量用户数据到本地数据库
- 调用场景: 定时任务(如每天凌晨同步一次)

**类型2: 实时登录认证接口**
- 接口: 移动端 `get_user_code_by_token` + `get_user_info`
- 特点: 单用户实时查询, OAuth流程
- 用途: 用户通过青橄榄移动端跳转登录时验证身份
- 调用场景: 用户访问系统时实时调用

**类型3: 管理员验证接口**
- 接口: 管理端 `verify_admin_user`
- 特点: Token验证, 获取管理员信息
- 用途: 管理员通过青橄榄管理端登录时验证身份
- 调用场景: 管理员访问系统管理功能时调用

## 三、关键问题与假设

### 3.1 接口集成架构问题

**问题1: 信息中心API与SSO登录流程的关系?**
- 信息中心API是否替代现有SSO登录流程?
- 还是作为补充, 用于用户数据预同步?

**假设A (并行使用):**
```
定时任务: 信息中心API → 批量同步用户数据到本地
用户登录: 青橄榄SSO → 验证身份 → 关联本地用户
```

**假设B (仅用信息中心):**
```
只使用信息中心API同步用户
不使用SSO登录流程
用户使用本地账号密码登录
```

**问题2: 现有SSO实现是否需要调整?**
- 当前代码实现了移动端和管理端SSO
- 但缺少真实token无法完整测试
- 是否需要保留这些实现?


### 3.2 数据一致性问题

**问题3: 如何保证数据同步与实时登录的一致性?**
- 如果用户在信息中心更新了手机号
- 定时同步前用户通过SSO登录
- 应该使用哪个数据源的信息?

## 四、建议的集成方案

### 方案A: 双轨并行(推荐)

**架构:**
```
1. 定时任务(每日凌晨): 信息中心API → 全量同步 → 本地user表
2. 用户登录: 青橄榄SSO → 验证 → 查询本地user表 → 创建session
3. 数据优先级: SSO实时数据 > 本地同步数据
```

**优点:**
- 本地有完整用户数据, 支持离线查询
- 登录时使用实时SSO验证, 数据最新
- 两套系统互为备份

**实现:**
- 新增Django management command: `sync_users_from_infocenter`
- 添加cron任务定期执行
- SSOUserMapping保留, 用于SSO登录时关联

---

### 方案B: 仅SSO实时认证

**架构:**
```
用户登录: 青橄榄SSO → 验证 → 获取用户信息 → 创建/更新本地user → session
不使用信息中心批量同步
```

**优点:**
- 实现简单
- 数据永远是最新的

**缺点:**
- 首次登录才创建本地用户
- 无法提前知道所有用户
- 管理员无法预先分配权限

---

### 方案C: 仅信息中心同步

**架构:**
```
1. 定时同步: 信息中心API → 全量同步 → 本地user表
2. 用户登录: 本地账号密码验证
3. 不使用青橄榄SSO
```

**优点:**
- 实现最简单
- 完全独立, 不依赖青橄榄在线

**缺点:**
- 失去SSO单点登录能力
- 用户需要额外记住本系统密码
- 与项目"青橄榄对接"需求不符

import hashlib
from typing import Literal


def generate_sign(
    app_secret: str,
    timestamp: str,
    rand_str: str,
    encryption_type: Literal['sha1', 'md5'] = 'sha1'
) -> str:
    """
    生成API签名

    签名规则：
    1. 将appSecret, timestamp, randStr三个参数值进行字典排序
    2. 拼接成字符串
    3. 进行sha1或md5加密

    Args:
        app_secret: AppSecret密钥
        timestamp: Unix时间戳（字符串）
        rand_str: 随机字符串
        encryption_type: 加密类型，'sha1'或'md5'，默认'sha1'

    Returns:
        签名字符串（十六进制小写）
    """
    if encryption_type not in ('sha1', 'md5'):
        raise ValueError(f"encryption_type must be 'sha1' or 'md5', got '{encryp
tion_type}'")

    # 1. 字典排序
    params = [app_secret, timestamp, rand_str]
    params.sort()

    # 2. 拼接
    sign_str = ''.join(params)

    # 3. 加密
    if encryption_type == 'md5':
        return hashlib.md5(sign_str.encode('utf-8')).hexdigest()
    else:
        return hashlib.sha1(sign_str.encode('utf-8')).hexdigest()
