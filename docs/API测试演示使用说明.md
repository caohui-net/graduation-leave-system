# API测试演示文档使用说明

本文档说明如何使用API测试演示文档作为用户操作手册的功能演示补充。

## 背景

**原始需求：** 创建WORD格式用户操作手册，包含用户操作界面截图

**实际交付：**
1. `docs/用户操作手册.docx` - WORD格式用户手册（含截图占位符）
2. `docs/API测试流程演示.md` - API测试流程完整演示文档

## API测试文档内容

`docs/API测试流程演示.md` 包含：

- **完整审批流程演示**（10个步骤）
  - 学生登录 → 提交申请 → 查询状态
  - 宿管员登录 → 查看待审批 → 审批通过
  - 辅导员登录 → 查看待审批 → 审批通过
  - 学生查看最终状态

- **审批驳回流程**
- **多宿管员协同审批演示**
- **API端点总结**
- **状态说明**

每个步骤包含：
- curl命令示例
- 完整JSON请求/响应
- 操作说明

## 使用方式

### 方案A：作为独立技术文档

将 `API测试流程演示.md` 作为技术文档，与用户手册并列使用：
- 用户手册：面向最终用户（学生/宿管员/辅导员）
- API测试文档：面向开发/测试/技术人员

### 方案B：补充到用户手册

在 `用户操作手册.docx` 末尾添加附录章节：
```
## 附录：API测试演示

由于微信小程序UI截图需要图形界面工具，本手册提供API测试流程演示作为功能验证参考。

详见：docs/API测试流程演示.md

该文档包含完整的curl命令和JSON响应示例，演示了从学生提交申请到辅导员审批通过的完整流程。
```

### 方案C：转换为截图

使用终端截图工具（如 `script` 命令或截图软件）：
1. 在终端执行API测试文档中的curl命令
2. 截取终端输出（命令+响应）
3. 插入到用户手册对应位置

虽然不是UI截图，但展示了系统真实功能。

## 文件位置

- 用户手册（WORD）：`docs/用户操作手册.docx`
- 用户手册（Markdown）：`docs/用户操作手册.md`
- API测试演示：`docs/API测试流程演示.md`
- 截图获取指南：`docs/截图获取说明.md`

## 建议

**推荐使用方案A或B：**
- 用户手册保持面向最终用户的叙述风格
- API测试文档作为技术补充材料
- 两者结合提供完整的系统说明

如果必须将API示例插入用户手册，建议：
- 简化JSON输出（仅保留关键字段）
- 添加注释说明
- 使用代码块格式化
