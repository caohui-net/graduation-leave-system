## 发现的问题

### P0-1: 测试脚本判定逻辑缺陷

**问题描述：**
`tests/multi_role_test.py` 在角色功能访问失败后仍将 `result.success` 置为 `true`

**影响：**
- 多角色5/5通过结论实际只代表登录成功
- 不能代表权限功能通过
- 误导性结论

**位置：**
```python
# tests/multi_role_test.py:1904
result["success"] = True  # 即使前面功能访问失败，仍置为True
```

**修复方案：**
只有当所有步骤（登录+功能访问）都通过时才设置 `success=True`

---

### P0-2: 审批端点URL错误

**问题描述：**
辅导员和宿管测试使用错误端点 `/api/approvals/pending/`

**实际端点：**
`/api/approvals/` (backend/apps/approvals/urls.py:846)

**影响：**
- 辅导员和宿管审批列表访问测试返回404
- 权限验证未被有效测试

**修复方案：**
更新测试脚本使用正确端点：
```python
# 错误
resp = requests.get(f"{BASE_URL}/api/approvals/pending/", headers=headers)

# 正确
resp = requests.get(f"{BASE_URL}/api/approvals/", headers=headers)
```

---

### P0-3: 完整审批流程未覆盖

**问题描述：**
第一阶段5轮申请均停在 `pending_dorm_manager` 状态

**缺失验证：**
1. 宿管审批操作
2. 辅导员审批操作
3. 状态流转（pending_dorm_manager → pending_counselor → approved）
4. 审批权限隔离

**影响：**
- 无法验证系统核心审批流程
- 状态机流转未测试
- 权限控制未验证

**修复方案：**
新增完整审批流程测试：
1. 学生提交申请
2. 宿管登录并审批
3. 验证状态变为 pending_counselor
4. 辅导员登录并审批
5. 验证状态变为 approved

---

### P1-4: 管理员403问题

**问题描述：**
`backend/apps/applications/views.py` 的 `list_applications` 未处理 `UserRole.ADMIN`

**代码位置：**
```python
# backend/apps/applications/views.py:949-952
else:
    return Response(
        {'error': {'code': 'FORBIDDEN', 'message': '无效的用户角色'}},
        status=status.HTTP_403_FORBIDDEN
    )
```

**影响范围：**
- 管理员无法访问 `/api/applications/` 列表
- `backend/apps/applications/permissions.py:can_view_application` 也未处理 ADMIN

**是否阻塞发布：**
- 若MVP包含管理员查看/管理申请功能：**阻塞发布**
- 若管理员功能不纳入MVP：**不阻塞，但需显式排除并记录风险**

---

### P2-5: 报告自相矛盾

**问题描述：**
综合报告同时声明：

**影响：**

2. 未验证但影响发布的范围
3. 发布阻塞条件
