# 浏览器缓存强制更新方案

## 问题
用户浏览器缓存旧版HTML/JS文件，导致SSO修复不生效。

## 方案1：回调URL版本号（推荐 ⭐⭐⭐⭐⭐）

### 操作步骤

**1. 登录青橄榄管理后台**

**2. 修改回调URL（添加版本参数）**

**移动端应用：**
```
旧：http://218.75.196.218:7788/mobile-sso-callback.html
新：http://218.75.196.218:7788/mobile-sso-callback.html?v=20260612
```

**管理端应用：**
```
旧：http://218.75.196.218:7788/admin-sso-callback.html
新：http://218.75.196.218:7788/admin-sso-callback.html?v=20260612
```

### 优势
- ✅ 立即生效，无需用户操作
- ✅ 强制浏览器加载新版本
- ✅ 对用户透明
- ✅ 后续更新只需修改版本号

### 版本号规则
- 日期格式：`v=YYYYMMDD`（如：v=20260612）
- 每次重大更新修改版本号
- 示例：`?v=20260612_2`（同一天多次更新）

## 方案2：HTML添加Cache-Control（辅助）

修改关键HTML文件，添加禁用缓存的meta标签。

### 文件清单

已添加Cache-Control meta标签：
- ✅ mobile-sso-callback.html
- ✅ admin-sso-callback.html
- ✅ sso-receiver.html

已更新版本号：
- ✅ index.html → js/api.js?v=20260612

### Meta标签说明
```html
<meta http-equiv="Cache-Control" content="no-cache, no-store, must-revalidate">
<meta http-equiv="Pragma" content="no-cache">
<meta http-equiv="Expires" content="0">
```

**作用：**
- `Cache-Control: no-cache` - 每次请求验证
- `no-store` - 不存储缓存
- `must-revalidate` - 强制重新验证
- `Pragma: no-cache` - HTTP/1.0兼容
- `Expires: 0` - 立即过期

## 方案3：用户手动清除（备用）

### 移动端浏览器
```
设置 → 清除浏览数据 → 勾选"缓存文件" → 清除
```

### 微信内置浏览器
```
我 → 设置 → 通用 → 存储空间 → 缓存 → 清理
```

### PC浏览器
- Chrome: `Ctrl+Shift+Delete` → 清除缓存
- 强制刷新: `Ctrl+F5` 或 `Ctrl+Shift+R`

### 版本管理建议

**创建版本更新脚本：**
```bash
#!/bin/bash
# update-version.sh

VERSION=$(date +%Y%m%d)

# 更新index.html中的JS版本号
sed -i "s/api\.js?v=[0-9-]*/api.js?v=$VERSION/" demo-web/index.html

echo "✅ 版本已更新为: v=$VERSION"
echo "⚠️  记得修改青橄榄回调URL：?v=$VERSION"
```
