# 配置中心管理指南

**版本**: v1.0  
**方案**: django-environ + Git  
**最后更新**: 2026-06-27

---

## 概述

统一管理三环境配置（dev/staging/production），消除配置漂移和人工错误。

**核心原则**:
- 配置模板化 - 统一字段定义
- 环境隔离 - 独立配置文件
- 版本控制 - Git管理变更历史
- 自动校验 - 发布前强制验证

---

## 文件结构

```
backend/
├── .env.template        # 配置模板（所有字段定义）
├── .env.dev            # 开发环境配置
├── .env.staging        # 预发布环境配置
├── .env.production     # 生产环境配置（敏感信息用占位符）
└── scripts/
    └── validate-config.py  # 配置校验脚本
```

**Git管理**:
- `.env.template` - 纳入版本控制
- `.env.dev` - 可选纳入（无敏感信息）
- `.env.staging` / `.env.production` - **不纳入**，仅占位符版本

## 配置模板

### 字段分类

#### 1. 数据库配置（必需）
```env
DB_HOST=localhost
DB_NAME=graduation_dev
DB_USER=app_dev_user
DB_PASSWORD=<password>
DB_PORT=5432
```

#### 2. Django核心配置（必需）
```env
SECRET_KEY=<50+ chars random string>
DEBUG=false
ALLOWED_HOSTS=localhost,127.0.0.1
```

#### 3. 功能开关（Feature Flags）
```env
ENABLE_STAY_SCHOOL=false
ENABLE_LEAVE_REQUEST=false
SSO_ENABLED=false
XG_API_ENABLED=false
```

#### 4. 外部服务配置（条件必需）
```env
# SSO配置（当SSO_ENABLED=true时必需）
SSO_QGL_APP_ID=
SSO_QGL_APP_SECRET=
SSO_QGL_BASE_URL=
SSO_CALLBACK_URL=

# XG-API配置（当XG_API_ENABLED=true时必需）
XG_API_BASE_URL=
XG_API_TENANT_CODE=
XG_API_CLIENT_ID=
XG_API_CLIENT_SECRET=
```

## 使用方法

### 1. 初始化配置

**开发环境**:
```bash
cd backend
cp .env.template .env.dev
vim .env.dev  # 填写开发环境配置
```

**Staging/Production**:
```bash
cp .env.template .env.staging
python scripts/validate-config.py .env.staging
```

### 2. 配置校验

**本地校验**:
```bash
python scripts/validate-config.py .env.production
```

**CI集成**:
```yaml
# .github/workflows/deployment-check.yml
- name: Validate production config
  run: |
    python backend/scripts/validate-config.py backend/.env.production.example
```

### 3. 应用配置

**Django设置集成**:
```python
# config/settings/base.py
import environ

env = environ.Env()
environ.Env.read_env('.env.{}'.format(os.getenv('DJANGO_ENV', 'dev')))

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': env('DB_NAME'),
        'USER': env('DB_USER'),
        'PASSWORD': env('DB_PASSWORD'),
        'HOST': env('DB_HOST'),
        'PORT': env('DB_PORT'),
    }
}
```

## 配置校验

### 自动检查项

1. **必需字段** - DB_HOST, DB_NAME, SECRET_KEY等
2. **条件必需** - 根据feature flag检查关联字段
3. **安全检查** - 密码长度、DEBUG状态、SECRET_KEY强度
4. **格式验证** - URL格式、端口号范围

## 环境配置标准

### 开发环境 (.env.dev)
```env
DEBUG=true
DB_HOST=localhost
ALLOWED_HOSTS=*
LOG_LEVEL=DEBUG
ENABLE_STAY_SCHOOL=true  # 本地测试全开
```

### Staging环境 (.env.staging)
```env
DEBUG=false
DB_HOST=staging-db.internal
ALLOWED_HOSTS=staging.example.com
LOG_LEVEL=INFO
ENABLE_STAY_SCHOOL=true  # 验证新功能
```

### 生产环境 (.env.production)
```env
DEBUG=false
DB_HOST=prod-db.internal
DB_USER=app_prod_user  # 受限权限账号
ALLOWED_HOSTS=app.example.com
LOG_LEVEL=WARNING
ENABLE_STAY_SCHOOL=false  # 灰度发布控制
```

## 敏感信息处理

### 1. 本地开发
- `.env.dev` 可以包含真实配置（不要有生产密码）
- 添加到 `.gitignore`

### 2. Staging/Production
- 真实密码**不纳入Git**
- Git中存放占位符版本 (`.env.production.example`)
- 实际部署时从密钥管理工具注入

### 3. 密钥管理工具集成（可选）

**AWS Secrets Manager**:
```bash
aws secretsmanager get-secret-value --secret-id prod/db/password \
  --query SecretString --output text > /tmp/db_password

export DB_PASSWORD=$(cat /tmp/db_password)
```

**Vault**:
```bash
vault kv get -field=password secret/prod/database
```

## 配置变更流程

### 1. 新增配置项
```bash
# 1. 更新模板
vim .env.template  # 添加新字段+注释

# 2. 更新校验脚本（如果是必需字段）
vim scripts/validate-config.py  # 添加到REQUIRED_FIELDS

# 3. 更新所有环境配置
vim .env.dev
vim .env.staging.example
vim .env.production.example

# 4. 文档化
# 更新本文档"字段分类"章节
```

### 2. 修改现有配置
```bash
# 1. 提交PR，说明变更原因
# 2. CR审查
# 3. 合并后通知运维更新实际环境配置
```

### 3. 删除废弃配置
```bash
# 1. 确认代码中无引用
grep -r "OLD_CONFIG_KEY" backend/

# 2. 从模板和环境配置中移除
# 3. 记录到CHANGELOG
```

## 集成到发布流程

### 发布前检查
```bash
# 部署检查清单
- [ ] 配置校验通过
- [ ] 敏感信息已从密钥工具获取
- [ ] 环境变量已设置（DJANGO_ENV=production）
```

### 部署脚本集成
```bash
#!/bin/bash
# deploy-production.sh

# 1. 校验配置
python scripts/validate-config.py .env.production || exit 1

# 2. 加载配置
set -a
source .env.production
set +a

# 3. 部署应用
systemctl restart graduation-backend-prod
```

## 故障排查

### 问题1: 配置未生效
```bash
# 检查环境变量是否正确加载
python manage.py shell
>>> import os
>>> os.getenv('DB_HOST')
```

### 问题2: 校验失败但字段已填写
```bash
# 检查是否有空格或引号问题
cat -A .env.production | grep DB_PASSWORD
# 正确: DB_PASSWORD=mypassword
# 错误: DB_PASSWORD= mypassword (前面有空格)
```

### 问题3: 多环境配置混乱
```bash
# 使用DJANGO_ENV环境变量明确指定
export DJANGO_ENV=production
python manage.py runserver
```

## 最佳实践

### 1. 配置分层
```
.env.template (模板)
  ├─ .env.dev (开发)
  ├─ .env.staging (预发布)
  └─ .env.production (生产)
```

### 2. 密码强度
- DB_PASSWORD: 至少16字符
- SECRET_KEY: 至少50字符，包含大小写/数字/特殊字符
- API密钥: 使用服务商生成的强随机值

### 3. 定期审计
```bash
# 每季度检查：
- [ ] 密码是否需要轮换
- [ ] 废弃配置项是否已清理
- [ ] 环境配置是否一致（除敏感信息外）
```

### 4. 版本控制策略
```gitignore
# .gitignore
.env.dev
.env.staging
.env.production

# 仅占位符版本纳入Git
!.env.*.example
```
