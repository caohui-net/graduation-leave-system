# 数据库备份配置

**配置日期**: 2026-06-27

## 备份策略

### 1. 全量备份（方案A）
- **频率**: 每日凌晨3点
- **保留**: 最近7天
- **格式**: SQL.gz压缩
- **位置**: `~/backup/graduation-leave-system/production_YYYYMMDD_HHMMSS.sql.gz`
- **脚本**: `/usr/local/bin/backup_graduation_db.sh`

### 2. WAL归档备份（方案B - PITR）
- **WAL归档**: 持续自动（16MB或30分钟触发）
- **归档目录**: `~/backup/graduation-leave-system/wal/`
- **基础备份**: 每周日凌晨2点
- **基础备份目录**: `~/backup/graduation-leave-system/base/`
- **保留策略**: 基础备份保留4周，WAL文件需手动清理
- **脚本**: `/usr/local/bin/base_backup_graduation_db.sh`

## 恢复能力

- **RPO（数据丢失）**: < 30分钟
- **RTO（恢复时间）**: 取决于数据量
- **PITR**: 支持任意时间点恢复

## PostgreSQL配置

```yaml
# docker-compose.prod.yml
volumes:
  - postgres_data:/var/lib/postgresql/data
  - /home/caohui/backup/graduation-leave-system/wal:/backup/wal
```

```conf
# postgresql.conf
wal_level = replica
archive_mode = on
archive_command = 'test ! -f /backup/wal/%f && cp %p /backup/wal/%f'
archive_timeout = 1800
```

## Cron配置

```bash
# 每日全量备份
0 3 * * * /usr/local/bin/backup_graduation_db.sh

# 每周基础备份
0 2 * * 0 /usr/local/bin/base_backup_graduation_db.sh
```

## 恢复流程

### 从全量备份恢复
```bash
gunzip < production_20260627_125450.sql.gz | docker exec -i production-db-1 psql -U postgres -d graduation_leave
```

### 从WAL归档恢复（PITR）
1. 恢复基础备份
2. 配置recovery.conf指定WAL归档位置
3. 重启PostgreSQL自动应用WAL日志
4. 恢复到指定时间点
