## 问题一：管理员后台缺少统计显示

### 问题描述
管理员用户访问后台页面（`demo-web/admin/dashboard.html`）时，无法看到审批统计数据（总审批数、待审批、已通过、已驳回）。

### 问题分析
1. 后台页面只显示用户信息和Token，没有统计卡片UI
2. 后端统计API已存在（`/api/approvals/statistics/`）且支持admin角色
3. 需要前端调用API并展示数据

### 解决方案
**文件**: `demo-web/admin/dashboard.html`

1. **添加统计卡片HTML**（第164-180行）：
   - 4个统计卡片：总审批数、待审批、已通过、已驳回
   - 使用grid布局

2. **添加CSS样式**（第105-135行）：
   - 响应式布局：手机2列（<768px），桌面4列（≥768px）
   - 渐变色背景，卡片阴影效果

3. **添加数据加载逻辑**（第216-248行）：
   - `loadStatistics()` 函数调用统计API
   - 页面加载时自动获取数据
   - 添加错误处理和调试日志

## 问题二：移动端统计卡片显示不全

### 问题描述
管理员在手机上访问后台时，4个统计卡片无法完整显示，第3-4个卡片被挤出视口或换行。

### 问题分析
使用 `grid-template-columns: repeat(auto-fit, minmax(200px, 1fr))` 会导致小屏幕上卡片过宽。

### 解决方案
**文件**: `demo-web/admin/dashboard.html`（第105-114行）

修改为响应式布局：
```css
.stats-grid {
  grid-template-columns: repeat(2, 1fr);  /* 手机2列 */
}
@media (min-width: 768px) {
  .stats-grid {
    grid-template-columns: repeat(4, 1fr);  /* 桌面4列 */
  }
}
```

## 问题三：审批列表筛选按钮不足

### 问题描述
PC端审批列表（`demo-web/index.html`）只有3个筛选按钮：全部、待审批、已审批。
缺少"已通过"和"已驳回"的独立筛选。

### 问题分析
1. 原设计将"已审批"作为"已通过+已驳回"的统称
2. 后端API支持单独筛选 `decision=approved` 和 `decision=rejected`
3. 用户需要独立查看"已通过"和"已驳回"的审批记录

### 解决方案
**文件**: `demo-web/index.html`

1. **修改筛选按钮HTML**（第1159-1164行）：
   ```html
   <div class="nav-tab active" onclick="filterApprovals('all')">全部</div>
   <div class="nav-tab" onclick="filterApprovals('pending')">待审批</div>
   <div class="nav-tab" onclick="filterApprovals('approved')">已通过</div>
   <div class="nav-tab" onclick="filterApprovals('rejected')">已驳回</div>
   ```

2. **更新筛选逻辑**（第254-273行）：
   - 修改 `filterApprovals()` 函数支持4个状态
   - 更新tab高亮逻辑

3. **修复API调用**（第72-82行）：
   - 移除前端二次过滤
   - 直接使用API返回的decision筛选结果

## 问题五：移动端筛选按钮超出屏幕

### 问题描述
手机上4个筛选按钮（全部、待审批、已通过、已驳回）无法完整显示在一行，只能看到前2个。

### 问题分析
1. 原始字号14px，padding 12px，4个按钮总宽度超出小屏幕
2. 中文文字"已通过"、"已驳回"占用空间较大
3. 需要压缩字号和padding

### 解决方案（第1次迭代）
**文件**: `demo-web/index.html`（第936-956行）

添加响应式缩小：
```css
@media (max-width: 480px) {
  .nav-tab { font-size: 11px; padding: 10px 4px; }
}
@media (max-width: 360px) {
  .nav-tab { font-size: 10px; padding: 8px 2px; }
}
```

### 解决方案（第2次迭代 - 最终）
**文件**: `demo-web/index.html`（第936-967行）

进一步优化：
```css
.nav-tab {
  padding: 10px 2px;
  font-size: 13px;
  border: 1px solid #e0e0e0;
  background: #f8f8f8;
}
@media (max-width: 480px) {
  .nav-tab { font-size: 11px; padding: 8px 2px; }
}
@media (max-width: 360px) {
  .nav-tab { font-size: 10px; padding: 8px 1px; }
}
.nav-tab.active {
  color: white;
  background: var(--primary-color);
  border-color: var(--primary-color);
}
```

**改进点**：
- 添加边框和背景色，视觉上更明显区分
- active状态使用蓝色背景+白色文字（移除底部下划线）
- 最小字号10px，最小padding 8px 1px

## 待优化项

1. **API错误处理**：
   - 当前只在Console输出错误，用户无感知
   - 建议添加Toast提示或错误占位符

2. **数据刷新机制**：
   - 统计数据只在页面加载时获取一次
   - 建议添加定时刷新或手动刷新按钮

3. **筛选按钮文字**：
   - 移动端字号10px可能对老年用户不友好
   - 可考虑使用图标+文字或纯图标

4. **统计卡片交互**：
   - 点击统计卡片可直接跳转到对应筛选
   - 例如：点击"待审批(680)" → 自动切换到"待审批"tab
