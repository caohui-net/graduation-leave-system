# 留校审批系统代码审核报告

**版本**：v2.0.0-stay-school  
**审核日期**：2026-06-26  
**状态**：建议待实施（本轮不实施）

---

## 一、整体评估

系统整体可用，已完成首次生产发布，核心审批流程（学生提交 → 辅导员审批 → approved）经验证正常。以下建议为后续迭代优化方向，**不影响当前可用状态**。

---

## 二、代码质量问题

### 🔴 P0：架构痛点（最高优先级）

| 问题 | 位置 | 说明 |
|------|------|------|
| `index.html` 超 2400 行 | `demo-web/index.html` | 结构、样式、业务逻辑全部混杂，维护成本极高 |
| 大量 `innerHTML` 字符串拼接 | 同上 | 可读性差，XSS 风险敞口大，样式修改困难 |
| 全局状态变量散落 | `currentFilter`、`approvalOffset` 等 | 跨页面状态同步易出 BUG |

**建议**：
- 短期：将 `index.html` 中的 JS 按功能抽离（`approval.js`、`application.js`）
- 长期：引入轻量框架（Vue 3 + Vite 或 Alpine.js）替代原生 DOM 拼接

---

### 🟡 P1：工程规范

| 问题 | 建议 |
|------|------|
| 接口响应格式不统一 | 统一 `{ code, message, data }` 结构 |
| 前端错误处理粗糙（`alert`） | 替换为 Toast/Snackbar 提示 |
| 审批状态无枚举常量 | 定义 `ApprovalStatus` 枚举，禁止魔法字符串 |
| 缺少操作审计日志 | 记录：提交/审批/驳回/导出等操作，含操作人、时间、IP |

## 三、安全性问题

### 🔴 必须确认已覆盖（当前系统）

| 风险点 | 状态 | 说明 |
|--------|------|------|
| JWT 签名校验 | ✅ 已实现 | 后端用 DRF + SimpleJWT |
| 权限越权（学生查他人申请） | ✅ 已实现 | 后端 `can_view_application` 权限检查 |
| SQL 注入 | ✅ 已规避 | Django ORM 参数化查询 |
| 文件上传类型白名单 | ✅ 已实现 | `ALLOWED_PREVIEW_TYPES` 限制 |
| 密码明文存储 | ✅ 已使用 | Django `set_password()` bcrypt |

### 🟡 需加固（后续迭代）

| 风险点 | 建议 |
|--------|------|
| XSS：`innerHTML` 大量拼接 | 有 `escapeHtml` 函数但覆盖不完整，重构时改用框架数据绑定 |
| Token 存储位置 | 当前用 `localStorage`，建议生产阶段评估 HttpOnly Cookie |
| 辅导员只能审批本人班级申请 | 已有 `approver=user` 过滤，需确认100%覆盖所有接口 |

## 四、UI/UX 改进建议（Gemini 重点建议）

> 以下来自 Gemini 对 `demo-web` 界面的专项分析。

### 🔴 最值得优先改进

#### 1. Loading 与反馈机制
- **现状**：网络请求期间无明显加载状态，失败时用 `alert()`
- **建议**：引入骨架屏或全局 Loading 遮罩；将 `alert` 替换为顶部/底部 Toast

#### 2. 审批详情页时间线
- **现状**：审批记录展示为普通卡片列表
- **建议**：使用 **Timeline 时间轴**展示审批流转，每步显示操作人、时间、意见，当前节点高亮

#### 3. 信息层级优化（卡片设计）
- **现状**：卡片内字段堆叠，主次不分
- **建议**：主标题加粗（`#333`），次要信息缩小置灰（`#999`），非核心字段折叠或右下角显示

### 🟡 移动端体验

| 问题 | 建议 |
|------|------|
| 复选框热区 `20px`，手指难点 | 扩大父元素热区至 `44×44px` |
| 批量操作栏突然出现 | 改为底部固定浮层，加 slide-up 动画 |
| 附件操作按钮小屏易误触 | 图标化或上下堆叠，下载时显示进度状态 |
| 状态 Tag 缺乏辅助识别 | 增加图标（✅ 通过、⏳ 待办、❌ 驳回）|

### 🟢 进一步提升（可选）

| 方向 | 具体建议 |
|------|---------|
| 工作台 Dashboard | 顶部统计卡片：待审批数、已通过、已驳回、今日留校人数 |
| 新建申请表单 | 分步表单（填写信息 → 上传材料 → 确认提交），字数统计，必填标星 |
| 申请列表筛选 | 增加状态/学院/时间范围筛选，提供重置和导出入口 |
| 空状态设计 | 替换空表格为友好提示 + 快捷操作按钮 |
| 审批弹窗 | 驳回原因必填；通过/驳回按钮颜色区分（绿/红）；防重复点击 |
| 侧边导航 | 大屏改为左侧 Sidebar，面包屑定位当前页 |
