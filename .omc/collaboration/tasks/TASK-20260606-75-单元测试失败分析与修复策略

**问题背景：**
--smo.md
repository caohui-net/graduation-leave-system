# Task: 单元测试失败分析与修复策略

**问题背景：**
- Smoke test（端到端集成测试）完全通过：H1+H2+N2全部验证成功
- 后端单元测试失败：19个测试失败（10 failures + 9 errors）
- 最近变更：seed_data.py添加building/department字段持久化

**失败模式：**
1. **404 != 201错误（多个测试）**
   - test_conflict_duplicate_application
   - test_complete_application_flow
   - test_forbidden_access_other_student_application
   - test_counselor_rejection
   - test_dorm_manager_rejection
   - test_application_submitted_notification_visible_via_api
   
   症状：Application创建返回404而非201

2. **IntegrityError**
   - test_counselor_step_requires_pending_counselor_status
   - null value in created_at column

3. **TransactionManagementError（多个测试）**
   - test_download_forbidden
   - test_upload_forbidden_counselor
   - test_dry_run_mode

4. **测试基础设施问题**
   - database test_graduation_leave already exists
   - EOFError when reading line（无交互终端）

**关键证据：**
- Smoke test使用真实seed_data创建的用户（2020001/M001/T001/T002）- 全部通过
- Unit tests使用test fixtures - 全部失败
- 最近修改：seed_data.py现在要求students有building+department，counselors有department，dorm_managers有building

**疑问：**
1. Unit test fixtures是否缺少必需的building/department字段？
2. Application创建路由逻辑是否依赖这些字段（导致404）？
3. 是否应该修复unit tests的fixtures，还是调整路由逻辑的容错性？
4. TransactionManagementError是否与并行测试或test database cleanup有关？

**期望输出：**
- 根本原因分析（为什么smoke test过但unit tests失败）
- 修复策略建议（修改fixtures vs 调整代码逻辑 vs 两者皆需）
- 优先级评估（这些失败是否阻塞Phase 4-5完成）
