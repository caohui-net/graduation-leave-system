# 三层环境自动监测部署指南（Phase 1）

## 概述

实现测试→生产的只读监测机制，不自动同步，仅告警提醒。

## 部署步骤

### 1. 测试环境（172.17.12.196）

```bash
# 赋予执行权限
chmod +x /opt/graduation-leave-system-staging/scripts/mark-staging-ready.sh

# 测试完成后手动执行
cd /opt/graduation-leave-system-staging
./scripts/mark-staging-ready.sh
```

### 2. 生产环境（172.17.12.196）

```bash
# 复制监测脚本
sudo cp /opt/graduation-leave-system/scripts/staging-change-detector.sh /usr/local/bin/
sudo chmod +x /usr/local/bin/staging-change-detector.sh

# 创建日志文件
sudo touch /var/log/deployment-alerts.log
sudo chown caohui:caohui /var/log/deployment-alerts.log

# 配置SSH免密登录（如未配置）
ssh-copy-id caohui@172.17.12.196

# 添加cron任务（每5分钟检查一次）
crontab -e
# 添加：
*/5 * * * * /usr/local/bin/staging-change-detector.sh
```

### 3. 验证

```bash
# 测试环境标记变更
cd /opt/graduation-leave-system-staging
./scripts/mark-staging-ready.sh

# 等待5分钟后查看生产告警日志
tail -f /var/log/deployment-alerts.log
```

## 工作流程

1. **测试环境验证通过** → 执行 `mark-staging-ready.sh`
2. **生产cron每5分钟检查** → 对比commit差异
3. **发现差异** → 写入日志告警
4. **运维人员查看日志** → 手动执行 `scripts/196-promote-to-prod.sh`

## 安全保证

- ✅ 只读监测，不修改生产环境
- ✅ SSH超时5秒，避免阻塞
- ✅ 告警仅写入日志，不自动执行
- ✅ 保留196脚本作为唯一发布入口
