## Python执行规范

### ❌ 错误做法
```bash
python manage.py migrate          # 找不到python命令
python3 manage.py migrate         # 系统Python无Django
```

### ✅ 正确做法
```bash
# 方式1: 使用venv中的Python（推荐）
backend/venv/bin/python3 manage.py migrate

# 方式2: 激活venv后执行
cd backend
source venv/bin/activate
python manage.py migrate
deactivate  # 退出venv
```

### 验证Django安装
```bash
# 系统Python（无Django）
python3 -m django --version
# 输出: No module named django

# venv Python（有Django 5.0）
backend/venv/bin/python3 -m django --version
# 输出: 5.0
```

## 数据库操作规范

### PostgreSQL连接
```bash
# 基本连接
psql -h localhost -U postgres -d graduation_dev

# 带密码环境变量
PGPASSWORD=yourpass psql -h localhost -U postgres -d graduation_dev
```

### Django数据库操作
```bash
cd backend
source venv/bin/activate

# 查看migrations状态
python manage.py showmigrations

# 创建migration
python manage.py makemigrations

# 应用migration
python manage.py migrate

# 回滚migration
python manage.py migrate users 0008
```

### 激活/退出venv
```bash
# 激活
cd backend && source venv/bin/activate
# 提示符变为: (venv) user@host:~/backend$

# 验证激活
which python
# 输出: /home/caohui/projects/graduation-leave-system/backend/venv/bin/python

# 退出
deactivate
```

### 创建venv（如果不存在）
```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## 常见错误与解决

### 错误1: command not found
```bash
# 错误
python manage.py migrate
# bash: python: command not found

# 解决
python3 manage.py migrate  # 或使用venv/bin/python3
```

### 错误2: No module named 'django'
```bash
# 错误
python3 manage.py migrate
# ImportError: Couldn't import Django

# 解决
backend/venv/bin/python3 manage.py migrate
```

### 错误3: Migration冲突
```bash
# 错误
django.db.migrations.exceptions.NodeNotFoundError

# 解决
# 1. 检查migration文件编号
ls backend/apps/users/migrations/00*.py

# 2. 修正dependencies指向正确的父migration
# 3. 重新编号冲突的migration文件
```

### 错误4: psql连接失败
```bash
# 错误
psql: could not connect to server

# 检查
# 1. 数据库是否运行
systemctl status postgresql
# 或
docker ps | grep postgres

# 2. 连接参数是否正确
echo $DB_HOST $DB_NAME $DB_USER
```
