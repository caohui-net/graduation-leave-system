## 故障处理

### 同步服务失败

**症状**: 代码修改后staging未更新

**排查**:
```bash
# 检查同步服务状态
systemctl --user status sync-to-staging.service

# 查看同步日志
tail -100 /tmp/sync-to-staging.log

# 手动触发同步
~/scripts/sync-to-staging.sh
```

**修复**:
```bash
# 重启同步服务
systemctl --user restart sync-to-staging.service
```

### Staging服务异常

**症状**: 健康检查失败

**排查**:
```bash
# 查看容器状态
ssh caohui@172.17.12.196 "docker ps | grep staging"

# 查看容器日志
ssh caohui@172.17.12.196 "docker logs staging-graduation-backend --tail 100"

# 查看数据库连接
ssh caohui@172.17.12.196 "docker exec staging-graduation-db pg_isready"
```

**修复**:
```bash
# 重启staging环境
ssh caohui@172.17.12.196 "
cd /opt/graduation-leave-system/staging
docker compose restart
"
```

### 生产环境异常

**症状**: 服务无响应或错误

**应急措施**:
```bash
# 1. 立即回滚到最近备份
ssh caohui@172.17.12.196 "
cd /opt/graduation-leave-system/production
docker compose down
LATEST=\$(ls -t /opt/backups | head -1)
rsync -av --delete /opt/backups/\$LATEST/production/ ./
docker compose up -d
"

# 2. 验证服务恢复
curl http://172.17.12.196:7787/readyz
```
