### 问题2：引用不存在的 Postman 文件 [P2]

**位置：** `docs/数据对接说明文档.md:417`

**问题描述：**  
引用 `docs/templates/宿舍管理系统API.postman_collection.json`，但 `docs/templates/` 目录中不存在此文件。

**影响：**

### 问题3：API 示例使用 HTTP 但安全要求 HTTPS [P1]

**位置：**  
- `docs/数据对接说明文档.md:54` - 示例使用 `http://...`
- `docs/数据对接说明文档.md:289` - 要求 API 必须 HTTPS

**问题描述：**  
示例地址使用 HTTP，但安全要求明确要求 HTTPS，存在矛盾。

**影响：**  
对接方可能按 HTTP 实现，导致安全审查不通过。

**修复建议：**  
统一为 HTTPS。但需考虑：**用户是否具备 SSL 证书？如果用户不具备 SSL 证书，是否应该考虑 HTTP？**

---

### 问题4：认证方式描述不够收敛 [P2]

**位置：** `docs/数据对接说明文档.md:56`

**问题描述：**  
同时列出 API Key、Basic Auth、IP 白名单三种认证方式，未明确推荐组合。

**影响：**  
对接方不清楚应该实现哪种方式，可能导致对接失败。

**修复建议：**  
明确唯一推荐组合，例如：**HTTPS + API Key + IP 白名单**。Basic Auth 若非必须建议删除。

---

### 问题5：数据脱敏与请求参数冲突 [P1]

**位置：**  
- `docs/数据对接说明文档.md:67` - API 请求参数允许传身份证号
- `docs/数据对接说明文档.md:301` - 要求导出数据不包含身份证号

**问题描述：**  
API 接口允许传身份证号查询，但数据脱敏要求不包含身份证号，存在矛盾。

**影响：**  
对接方不清楚是否应该支持身份证号查询，可能导致隐私泄露风险。

**修复建议：**  
不要在宿舍清退接口中使用 `id_card`，用 `student_id + name` 足够。

---

### 问题6：审批时限口径不一致 [P2]

**位置：**  
- `docs/用户设计说明书.md:57` - 写"24小时内"
- 系统设计文档 - 倾向"1 个工作日，按 9:00-17:00 工作时间计算"

**问题描述：**  
用户文档与系统设计文档审批时限口径不一致。

**影响：**  
用户期望与系统实际行为不符，可能导致投诉。

**修复建议：**  
统一改为"1 个工作日"。

---

### 问题7：初始密码为身份证后 6 位有安全风险 [P0]

**位置：** `docs/用户设计说明书.md:84`

**问题描述：**  
初始密码为身份证后 6 位，弱口令风险高，且扩大身份证信息依赖。

**影响：**  
严重安全风险，容易被暴力破解。

**修复建议：**  
改为以下方案之一：
1. 首次登录强制设置密码
2. 临时随机密码（通过短信/邮箱发送）
3. 短信或邮箱验证码初始化

---

### 问题8：辅导员权限按"本年级"过粗 [P2]

**位置：**  
- `docs/用户设计说明书.md:115`
- `docs/用户设计说明书.md:143`

**问题描述：**  
辅导员查看/审批权限按"本年级"划分过粗，实际应按学院、专业、班级或辅导员负责范围授权。

**影响：**  
权限控制不够精细，可能导致越权访问。

**修复建议：**  
改为按辅导员负责的班级或专业授权。

---

### 问题9：容量指标与既有设计口径不一致 [P2]

**位置：**  
- `docs/用户设计说明书.md:178` - 在线用户 500
- `docs/用户设计说明书.md:179` - 并发请求 50、QPS 100、P95 < 500ms
- 系统设计文档 - P95 < 200ms、500 并发用户

**问题描述：**  
性能指标在不同文档中口径不一致。

**影响：**  
验收标准不明确，可能导致验收争议。

**修复建议：**  
统一性能指标为系统设计文档的口径：
- 在线用户：500
- 并发请求：50
- QPS：100
- P95 响应时间：< 200ms
