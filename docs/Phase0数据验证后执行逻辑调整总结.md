---

## 调整背景

Phase 0数据验证完成后，发现多项与原计划不符的数据情况：
1. 116名File2独有学生缺少楼栋数据（98% vs 100%覆盖率）
2. 271名研究生使用临时ID，已找到真实学号
3. 新增3名管理员（2学工管理员+1兜底宿管员）
4. 发现19名额外研究生不在源数据

## 已实施修改

### 1. 兜底宿管员路由逻辑（方案B）

**实施时间：** 2026-06-06  
**文件：** `backend/apps/applications/views.py`  
**提交：** feat: 实现兜底宿管员Fallback路由逻辑（方案B）

**修改内容：**
```python
# 原逻辑（单一路由）：
dorm_manager = User.objects.get(role=UserRole.DORM_MANAGER, building=user.building)

# 新逻辑（Fallback机制）：
1. 先按building查找宿管员
2. 如果building为空或找不到，使用兜底宿管员(职工号92008149)
3. 如果兜底宿管员也不存在，返回404错误
```

**业务影响：**
- 116名无楼栋学生可正常提交申请
- 路由到兜底宿管员程婷进行审批
- 不阻塞正常流程

**后续计划：**
- 短期：使用方案B（当前实施）
- 长期：升级到方案C（数据库标记位），增强可维护性
