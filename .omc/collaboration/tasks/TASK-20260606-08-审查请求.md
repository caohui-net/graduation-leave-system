### 问题2：271人学号更新策略

**方案A：数据库直接更新**
```python
# Django management command
python3 manage.py update_tmp_ids \
  --mapping backend/data/missing_student_no_filled.csv
```

**优点：** 快速，不影响其他数据  
**缺点：** 临时ID历史记录保留，可能混淆

**方案B：重新导入**
```bash
# 1. 更新File2学号
# 2. 重新合并File5
# 3. 重新导入所有学生
```

**优点：** 数据一致性好，无临时ID残留  
**缺点：** 重新导入可能影响已有申请记录

**审查要点：**
1. 推荐策略及理由
2. 数据一致性风险
3. 已有申请记录处理
4. 回滚方案

### 问题3：19名额外研究生处理
