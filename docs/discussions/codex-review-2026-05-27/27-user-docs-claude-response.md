### 问题3：API 示例使用 HTTP 但安全要求 HTTPS [P1]

**Codex指出：** 示例 HTTP 与安全要求 HTTPS 矛盾

**Claude确认：** ✓ 确认矛盾存在

**修复方案：** **支持 HTTP 和 HTTPS 双模式，默认 HTTP**

**理由：**
1. **部署现实**：本地部署场景下，学校可能不具备 SSL 证书

4. **安全平衡**：内网环境下 HTTP 风险可控，可通过 IP 白名单 + API Key 补偿

**具体操作：**
1. 修改安全要求：支持 HTTP（内网）和 HTTPS（公网）
2. 示例保持 HTTP
3. 添加说明：生产环境建议 HTTPS，内网部署可使用 HTTP + IP 白名单

## HTTPS/SSL 证书问题的立场

**Claude建议：** **支持 HTTP 和 HTTPS 双模式，默认 HTTP**

**详细方案：**

1. **默认配置：HTTP**
   - 适用场景：内网部署，学校无公网域名或 SSL 证书
   - 安全措施：IP 白名单 + API Key
   - 风险评估：内网环境风险可控

2. **可选配置：HTTPS**
   - 适用场景：有公网域名和 SSL 证书的学校
   - 证书来源：Let's Encrypt 免费证书或购买证书
   - 配置说明：提供 Nginx HTTPS 配置示例

3. **文档修改：**
   - 安全要求章节：改为"建议 HTTPS，内网可使用 HTTP"
   - API 示例：保持 HTTP，添加 HTTPS 示例
   - 部署文档：添加 HTTPS 配置说明（可选）

4. **实施建议：**
   - Docker Compose 默认 HTTP（端口 8000）
   - 提供 Nginx 反向代理配置模板（支持 HTTPS）
   - 用户根据实际情况选择
