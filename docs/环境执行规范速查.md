## Python执行规范

### ❌ 错误做法
```bash
python manage.py migrate          # 找不到python命令
python3 manage.py migrate         # 系统Python无Django
```

### ✅ 正确做法
```bash
# 方式1: 使用venv中的Python（推荐）
backend/venv/bin/python3 manage.py migrate

# 方式2: 激活venv后执行
cd backend
source venv/bin/activate
python manage.py migrate
deactivate  # 退出venv
```

### 验证Django安装
```bash
# 系统Python（无Django）
python3 -m django --version
# 输出: No module named django

# venv Python（有Django 5.0）
backend/venv/bin/python3 -m django --version
# 输出: 5.0
```

## 常见错误与解决

### 错误1: command not found
```bash
# 错误
python manage.py migrate
# bash: python: command not found

# 解决
python3 manage.py migrate  # 或使用venv/bin/python3
```

### 错误2: No module named 'django'
```bash
# 错误
python3 manage.py migrate
# ImportError: Couldn't import Django

# 解决
backend/venv/bin/python3 manage.py migrate
```

### 错误3: Migration冲突
```bash
# 错误
django.db.migrations.exceptions.NodeNotFoundError

# 解决
# 1. 检查migration文件编号
ls backend/apps/users/migrations/00*.py

# 2. 修正dependencies指向正确的父migration
# 3. 重新编号冲突的migration文件
```

### 错误4: psql连接失败
```bash
# 错误
psql: could not connect to server

# 检查
# 1. 数据库是否运行
systemctl status postgresql
# 或
docker ps | grep postgres

# 2. 连接参数是否正确
echo $DB_HOST $DB_NAME $DB_USER
```

## 生产环境部署规范

### ❌ 错误做法
```bash
# 错误1: 本地执行部署脚本
bash scripts/196-promote-to-prod.sh
# ❌ 失败原因: 脚本需在196服务器执行

# 错误2: SSH后直接执行脚本
ssh caohui@172.17.12.196 "bash scripts/196-promote-to-prod.sh"
# ❌ 失败原因: 找不到脚本路径，需要cd到项目目录

# 错误3: 在错误目录执行
ssh caohui@172.17.12.196 "cd /home/caohui && bash scripts/196-promote-to-prod.sh"
# ❌ 失败原因: 脚本不在home目录
```

### ✅ 正确做法
```bash
# 完整部署命令（一行）
ssh caohui@172.17.12.196 "cd /opt/graduation-leave-system && bash scripts/196-promote-to-prod.sh"

# 或分步执行
ssh caohui@172.17.12.196
cd /opt/graduation-leave-system
bash scripts/196-promote-to-prod.sh
```

### 常见部署错误

#### 错误1: Staging未就绪
```bash
# 错误信息
❌ Staging未就绪

# 解决方案
# 1. 检查staging服务状态
ssh caohui@172.17.12.196 "docker ps | grep staging"

# 2. 启动staging服务
ssh caohui@172.17.12.196 "cd /opt/graduation-leave-system/staging && docker compose up -d"

# 3. 等待就绪后重试部署
```

#### 错误2: 数据库迁移失败
```bash
# 错误信息
❌ Migration failed

# 解决方案
# 1. 检查迁移文件冲突
ssh caohui@172.17.12.196 "cd /opt/graduation-leave-system/production && docker compose -f docker-compose.prod.yml exec backend python manage.py showmigrations"

# 2. 手动执行迁移
ssh caohui@172.17.12.196 "cd /opt/graduation-leave-system/production && docker compose -f docker-compose.prod.yml exec backend python manage.py migrate --fake-initial"

# 3. 必要时回滚
```

#### 错误3: 服务启动超时
```bash
# 错误信息
❌ 容器启动超时

# 解决方案
# 1. 查看容器日志
ssh caohui@172.17.12.196 "cd /opt/graduation-leave-system/production && docker compose -f docker-compose.prod.yml logs backend --tail 50"

# 2. 检查端口占用
ssh caohui@172.17.12.196 "netstat -tuln | grep 7788"

# 3. 重启服务
ssh caohui@172.17.12.196 "cd /opt/graduation-leave-system/production && docker compose -f docker-compose.prod.yml restart"
```
