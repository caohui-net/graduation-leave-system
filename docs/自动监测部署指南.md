# 三层环境自动监测部署指南（Phase 1）

## 概述

实现开发→staging→生产的只读监测机制，不自动同步，仅告警提醒。

## 环境架构

- **开发环境**: 172.17.12.199 (本地git仓库)
- **Staging/生产环境**: 172.17.12.196 (同一目录，docker区分端口)
  - Staging: 端口17788(前端)/17787(后端)
  - Production: 端口7788(前端)/7787(后端)

## 部署步骤（已完成）

### 1. SSH免密配置

```bash
# 196服务器 → 199开发机
ssh caohui@172.17.12.196 "ssh-keygen -t rsa -N '' -f ~/.ssh/id_rsa"
ssh caohui@172.17.12.196 "cat ~/.ssh/id_rsa.pub" >> ~/.ssh/authorized_keys
```

### 2. 部署监测脚本

```bash
# 从开发机复制脚本到196
scp scripts/{staging-change-detector,mark-staging-ready}.sh caohui@172.17.12.196:/opt/graduation-leave-system/scripts/

# 在196配置
ssh caohui@172.17.12.196 "
  chmod +x /opt/graduation-leave-system/scripts/*.sh
  sudo cp /opt/graduation-leave-system/scripts/staging-change-detector.sh /usr/local/bin/
  sudo chmod +x /usr/local/bin/staging-change-detector.sh
  sudo touch /var/log/deployment-alerts.log
  sudo chown caohui:caohui /var/log/deployment-alerts.log
"
```

### 3. 配置Cron

```bash
ssh caohui@172.17.12.196 "
  (crontab -l 2>/dev/null | grep -v staging-change-detector; echo '*/5 * * * * /usr/local/bin/staging-change-detector.sh') | crontab -
"
```

## 使用流程

### Staging验证通过后标记

```bash
ssh caohui@172.17.12.196 "/opt/graduation-leave-system/scripts/mark-staging-ready.sh"
```

### 查看监测日志

```bash
ssh caohui@172.17.12.196 "tail -f /var/log/deployment-alerts.log"
```

### 发布到生产

```bash
# 监测到差异后，手动执行
ssh caohui@172.17.12.196 "cd /opt/graduation-leave-system && scripts/196-promote-to-prod.sh"
```

## 工作流程

1. **开发机提交新代码** → staging容器验证
2. **验证通过** → 执行 `ssh caohui@172.17.12.196 "/opt/graduation-leave-system/scripts/mark-staging-ready.sh"`
3. **5分钟内cron检测到差异** → 写入 `/var/log/deployment-alerts.log`
4. **运维查看日志** → 手动执行 `scripts/196-promote-to-prod.sh`

## 安全保证

- ✅ 只读监测，不修改生产环境
- ✅ SSH超时5秒，避免阻塞
- ✅ 告警仅写入日志，不自动执行
- ✅ 保留196脚本作为唯一发布入口
