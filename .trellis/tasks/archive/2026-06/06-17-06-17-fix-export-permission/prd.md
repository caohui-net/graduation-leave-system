# 修复导出功能权限漏洞

## Goal

管理员导出功能应遵循权限规则，只能导出当前用户可查看的审批数据，不能导出全部数据。

## Requirements

**安全性问题：**
当前导出功能无权限过滤，所有管理员可导出全部数据，存在数据泄露风险。

**权限规则（需与列表查询一致）：**
- 宿管员：只能导出自己负责楼栋的申请
- 辅导员：只能导出自己负责班级的申请
- 学工部/管理员：可导出全部申请

## Acceptance Criteria

- [ ] 宿管员导出：仅包含自己负责楼栋的申请数据
- [ ] 辅导员导出：仅包含自己审批的申请数据
- [ ] 学工部导出：包含全部申请数据（不变）
- [ ] 导出逻辑与list_applications权限逻辑一致

## Implementation

**修改文件：**
- 查找导出API endpoint（可能在applications/views.py或单独export视图）
- 复用list_applications的queryset过滤逻辑
- 应用相同的role-based filtering

**测试：**
- 宿管账号导出，验证数据范围
- 辅导员账号导出，验证数据范围
- 学工部账号导出，验证数据完整性
