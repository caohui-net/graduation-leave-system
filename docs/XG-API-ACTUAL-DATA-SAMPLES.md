## 实际数据样例

### 样例1：学生用户记录（湖南工学院真实数据）

**API响应结构：** 注意与文档示例不同，实际使用 `users` 数组而非嵌套 `data.data`

```json
{
  "code": 200,
  "msg": "success",
  "current_page": 1,
  "per_page": "10",
  "total": 32039,
  "users": [
    {
      "id": 8925,
      "number": "2025110140314",
      "name": "乐绍钧",
      "phone": "15334282752",
      "status": 1,
      "parent_dep": [{"name": "黄冈师范学院", "level": 1}],
      "department": [{"name": "计算机学院", "level": 2}],
      "user_identity": {"id": 4, "name": "学生"}
    }
  ]
}
```

**字段提取验证：**

| XG字段 | 实际值 | 内部映射字段 | 映射后值 | 验证 |
|---|---|---|---|---|
| number | "2025110140314" | User.user_id | "2025110140314" | ✓ 直接映射 |
| name | "乐绍钧" | User.name | "乐绍钧" | ✓ 直接映射 |
| phone | "15334282752" | User.phone | "15334282752" | ✓ 直接映射 |
| status | 1 | User.active | True | ✓ 1→True映射 |
| department[0].name | "计算机学院" | User.department | "计算机学院" | ✓ 取第一个 |

### 发现的问题

**中优先级问题：**
- phone字段有20%为空字符串（非null），需映射器处理
- user关联对象10%缺失，需映射时容错
- API响应结构与文档不一致：实际用`users`数组，非`data.data`

**低优先级问题：**
- 客户端暂不支持过滤参数（name/number筛选）
- per_page返回字符串类型需转int
