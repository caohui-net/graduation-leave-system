# Task: Phase 3数据缺口：6个学院无辅导员（1957名学生）

**问题背景：**
运行validate_import.py发现严重数据质量问题：
- 6个学院在counselors_processed.csv中无辅导员数据
- 影响1,957名学生无法完成审批流程（约33%）
- 这是Phase 3数据导入的P0阻塞问题

**缺失学院和学生数：**
1. 建筑与工程学院: 230人
2. 文学院(苏东坡书院): 483人
3. 新闻与传播学院: 327人
4. 旅游文化与地理科学学院: 287人
5. 生命科学学院: 264人
6. 音乐与戏剧学院: 366人

**已确认事实：**
- grep counselors_processed.csv无这6个学院的记录
- 验证脚本sample testing显示40%路由失败率
- 现有20个辅导员覆盖21个学院（有重复，但缺这6个）

**需要讨论的方案：**
1. 是否有其他数据源可补充这6个学院的辅导员？
2. 是否可以将这些学生重新分配到现有辅导员（就近原则）？
3. 是否可以临时指定fallback辅导员（类似宿管员fallback机制）？
4. 是否需要联系业务方补充这6个学院的辅导员数据？

**业务影响：**
- 阻塞1,957名学生的毕业离校申请
- Phase 4/5无法推进（前端无意义，测试会失败）
- 需要在Phase 3.5解决此问题

**建议讨论参与者：** Codex（数据架构视角）, Gemini（业务方案建议）
