## 三、必需数据源（P0）

### 3.1 学生基本信息

**数据来源：** 教务系统

**用途：**
- 创建学生账号
- 身份验证
- 自动分配审批人（根据班级）

**必需字段：**
- 学号（student_id）
- 姓名（name）
- 院系（department）
- 专业（major）
- 班级（class_name）
- 年级（grade）
- 手机号（phone，可选）
- 邮箱（email，可选）

**对接方式：**
- API接口（推荐）
- CSV文件导入（备选）

**当前状态：** ❌ 未在数据对接文档中说明

---

### 3.2 宿舍清退数据

**数据来源：** 宿管系统

**用途：**
- 验证学生是否完成宿舍清退
- 作为离校申请的前置条件

**必需字段：**
- 学号（student_id）
- 姓名（name）
- 宿舍信息（dorm_building, dorm_room, bed_number）
- 清退状态（is_checked_out, checkout_date）

**对接方式：**
- API接口（推荐）
- CSV文件导入（备选）

**当前状态：** ✅ 已在数据对接文档中详细说明

---

### 3.3 辅导员基本信息

**数据来源：** 人事系统

**用途：**
- 创建辅导员账号
- 分配审批权限

**必需字段：**
- 工号（employee_id）
- 姓名（name）
- 手机号（phone）
- 邮箱（email）
- 院系（department，可选）
- 初始密码（password，或使用工号作为初始密码）

**对接方式：**
- API接口（推荐）
- CSV文件导入（备选）

**当前状态：** ❌ 未在数据对接文档中说明

---

### 3.4 班级-辅导员对应关系

**数据来源：** 教务系统

**用途：**
- 自动分配审批人
- 学生提交申请时，系统根据班级自动找到对应辅导员

**必需字段：**
- 班级名称（class_name）
- 辅导员工号（counselor_employee_id）
- 院系（department）
- 年级（grade）

**对接方式：**
- API接口（推荐）
- CSV文件导入（备选）

**当前状态：** ❌ 未在数据对接文档中说明

**重要性：** 🔴 极高
- 如果没有这个数据，系统无法自动分配审批人
- 需要手动配置每个学生的辅导员，工作量巨大

## 四、可选数据源（P1）

### 4.1 图书馆清书状态

**数据来源：** 图书馆系统

**用途：**
- 验证学生是否归还图书
- 作为离校申请的前置条件

**必需字段：**
- 学号（student_id）
- 姓名（name）
- 是否清书（is_cleared）
- 清书日期（clear_date）

**对接方式：**
- API接口（推荐）
- CSV文件导入（备选）

**当前方案：** 学生上传图书馆清书证明截图（手动验证）

**是否需要对接：** 可选
- 如果图书馆系统支持API，建议对接
- 如果不支持，继续使用手动上传方式

---

### 4.2 财务结清状态

**数据来源：** 财务系统

**用途：**
- 验证学生是否结清费用
- 作为离校申请的前置条件

**必需字段：**
- 学号（student_id）
- 姓名（name）
- 是否结清（is_cleared）
- 结清日期（clear_date）

**对接方式：**
- API接口（推荐）
- CSV文件导入（备选）

**当前方案：** 学生上传财务结清截图（手动验证）

**是否需要对接：** 可选
- 如果财务系统支持API，建议对接
- 如果不支持，继续使用手动上传方式

---

### 4.3 学籍状态（毕业生识别）

**数据来源：** 教务系统

**用途：**
- 自动识别毕业生
- 限制只有毕业生可以提交离校申请
- 自动设置计划离校日期范围

**必需字段：**
- 学号（student_id）
- 姓名（name）
- 学籍状态（status：在读/毕业/结业/肄业）
- 毕业时间（graduation_date）
- 年级（grade）

**对接方式：**
- API接口（推荐）
- 包含在学生基本信息中（备选）

**当前方案：** 可能未考虑

**是否需要对接：** 建议对接
- 防止非毕业生误操作
- 自动化程度更高
