## 2. 关键安全隐患

### 2.1 SSH免密登录配置（原方案 4.1.1）

**问题**:
```bash
ssh-keygen -t ed25519 -C "dev-to-prod-sync"
# 未指定 -N 参数，默认无密码保护
```

**风险**: 199服务器被入侵后，攻击者可直接访问196生产环境

**加固方案**:
```bash
# 1. 私钥添加密码保护
ssh-keygen -t ed25519 -N "$(openssl rand -base64 32)" -C "dev-to-prod-sync"

# 2. 限制来源IP（196的sshd_config）
Match User root Address 172.17.12.199
    PasswordAuthentication no
    PubkeyAuthentication yes
    AuthorizedKeysFile /root/.ssh/authorized_keys.dev2prod

# 3. 私钥权限加固
chmod 400 ~/.ssh/id_ed25519
chattr +i ~/.ssh/id_ed25519  # 防止意外删除
```

### 2.2 生产环境配置安全（原方案 4.3.2）

**问题**: `.env.prod` 明文存储敏感信息

```bash
# 当前方案
SECRET_KEY=<生产环境密钥>
DATABASE_URL=postgresql://postgres:prod_password@db:5432/graduation_leave
```

**风险**: 
- Git误提交泄露
- rsync同步到错误目标
- 开发人员直接查看

**加固方案**:
```bash
# 方案A: 使用加密配置文件
age --encrypt --recipient <pubkey> .env.prod > .env.prod.age
# 部署时解密：age --decrypt .env.prod.age > .env.prod

# 方案B: 引用外部密钥管理
# docker-compose.prod.yml
services:
  backend:
    env_file:
      - .env.prod
    secrets:
      - db_password
      - secret_key

secrets:
  db_password:
    external: true
  secret_key:
    external: true
```

### 2.3 自动同步代码审查缺失

**风险**: 恶意代码或bug直接进入生产环境

**当前方案**: inotify检测到变更 → 1秒后rsync → 生产立即生效

**问题场景**:
1. 开发人员误提交包含密钥的文件
2. 引入有安全漏洞的第三方库
3. 代码逻辑错误导致数据损坏

**加固方案**:
```bash
# 同步前钩子脚本
# /home/caohui/scripts/pre-sync-check.sh
#!/bin/bash

# 1. 检查敏感信息
if grep -r "SECRET_KEY\|PASSWORD\|API_KEY" "$SOURCE_DIR" --exclude-dir=.git; then
    echo "[BLOCKED] Sensitive data detected"
    exit 1
fi

# 2. 静态代码扫描（可选）
bandit -r "$SOURCE_DIR/backend" -f txt || echo "[WARN] Security issues found"

# 3. 语法检查
python -m py_compile "$SOURCE_DIR/backend"/**/*.py || exit 1

echo "[OK] Pre-sync checks passed"
```

## 3. 最小权限原则违背

**当前方案**: 196使用root用户运行所有服务

```yaml
# docker-compose.prod.yml
services:
  backend:
    user: root  # ❌ 不必要的高权限
```

**推荐**:
```yaml
services:
  backend:
    user: "1000:1000"  # 非root用户
    
  db:
    user: postgres
```

**systemd服务**:
```ini
# /etc/systemd/system/graduation-frontend-prod.service
[Service]
User=www-data  # 非root
Group=www-data
```
