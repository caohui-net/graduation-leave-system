# 数据库字段映射关系

## 核心模型

### User (用户表)
```
PK: user_id (学工号/工号)
Fields:
  - name: 姓名
  - role: student/dorm_manager/counselor/dean/admin
  - active: 激活状态
  - class_id: 班级ID (学生)
  - is_graduating: 应届标识 (学生)
  - graduation_year: 毕业年份 (学生)
  - phone: 手机号
  - email: 邮箱
  - department: 学院/部门
  - building: 宿舍楼栋 (学生)
  - campus: 校区 (学生)
  - room_number: 房间号 (学生)
  - gender: 性别
  - major: 专业 (学生)
  - level: 学历层次 (学生)
```

### Application (离校申请表)
```
PK: application_id
FK: student → User (PROTECT)

Fields:
  - student_name: 学生姓名 (冗余)
  - class_id: 班级ID (冗余)
  - contact_phone: 联系电话
  - reason: 离校原因
  - leave_date: 离校日期
  - status: draft/pending_dorm_manager/pending_counselor/approved/rejected/failed
  - dorm_checkout_status: completed/pending/not_started/unknown
  - created_at, updated_at

Constraint: 单个学生只能有1个active申请 (pending/approved)
```

### Approval (审批记录表)
```
PK: approval_id
FK: application → Application (PROTECT)
FK: approver → User (PROTECT) - 指定审批人
FK: decided_by → User (PROTECT, nullable) - 实际审批人 (代理)

Fields:
  - step: dorm_manager/counselor/dean
  - approver_name: 审批人姓名 (冗余)
  - decision: pending/approved/rejected
  - comment: 审批意见
  - decided_at: 审批时间
  - created_at, updated_at

每个申请有2条Approval记录 (宿管员 + 辅导员)
```

## 数据关系图

```
User (学生)
  ↓ 1:N (FK: student)
Application (离校申请)
  ↓ 1:2 (FK: application)
Approval (审批记录)
  ↓ N:1 (FK: approver)
User (宿管员/辅导员)
  ↓ N:1 (FK: decided_by, nullable)
User (实际审批人)
```

## 路由映射

### 宿管员路由
```python
student.building → dorm_manager
student.building == NULL → 默认宿管员(92008149)
```

### 辅导员路由
```python
student.department → counselor
# 通过学院名称匹配
```

## 冗余字段

**Application表冗余**:
- `student_name` = student.name
- `class_id` = student.class_id

**Approval表冗余**:
- `approver_name` = approver.name

**目的**: 减少JOIN查询，提升列表性能

## 约束规则

1. **User.user_id**: 唯一主键
2. **Application唯一性**: 同一学生只能有1个active申请
3. **外键保护**: 所有FK使用PROTECT防止误删
4. **审批记录数**: 每个申请固定2条Approval (宿管员+辅导员)
