# 数据处理流程

## 流程概览

```
毕业生CSV → 学号匹配 → 合并处理 → 最终导入CSV → 数据库
在校生Excel → class_id修复 → 直接导入 → 数据库
教职工CSV → 标准化 → 导入 → 数据库
```

## 详细步骤

### 步骤1: 学号匹配

**输入**:
- `1-5830名毕业生（含研究生）.csv` - 含姓名、宿舍，缺学号

**处理**:
- 通过姓名从其他数据源匹配学号
- 生成 `missing_student_no_filled.csv` (31KB) - 匹配结果

**输出**:
- 补全学号的学生记录

### 步骤2: 数据合并

**输入**:
- 原始毕业生CSV（宿舍信息）
- 学号匹配结果

**处理**:
- 合并字段: 学号 + 姓名 + 班级 + 宿舍信息
- 添加固定字段: is_graduating=true, graduation_year=2026

**输出**:
- `file5_students_merged_v2.csv` (788KB) - 最终导入文件

### 步骤3: 在校生导入

**输入**:
- `15975名在校生（不含毕业生）.xls` (Excel 97-2003格式)

**处理**:
- 使用pandas读取（engine='xlrd'）
- 字段映射: 学号→student_no, 姓名→name, 班级→class_id
- 标识字段: is_graduating=null（区分在校生）

**输出**:
- 直接导入数据库: 15,975条记录

**注意**:
- 602条记录班级为空（占3.8%）
- 导入后需修复class_id（见步骤5）

### 步骤4: 辅导员处理

**输入**:
- `2026年学院辅导员信息统计表.csv`
- `2026年社区辅导员信息统计表.csv`

**处理**:
- 合并学院+社区辅导员
- 标准化格式: employee_id, name

**输出**:
- `counselors_processed.csv` (1.3KB)

### 步骤5: class_id字段修复（2026-06-29）

**背景**:
- 导入后发现21,940条记录中891条class_id为空
- 毕业生和在校生数据源的班级字段缺失

**处理**:
```sql
-- 修复脚本见：backend/scripts/fix_class_id.py
-- 从原始Excel重新读取班级字段
-- 按学号匹配更新
```

**结果**:
- 修复前: 21,049条有效 (95.9%)
- 修复后: 21,049条有效 (891条确认源数据无班级)
- 详细记录: `docs/class_id字段修复记录.md`

## 数据关系

```
辅导员 ──1:N── 班级映射 ──1:N── 学生（毕业生+在校生）
   ↓                          ↓
counselors_processed    file5_students_merged_v2 (毕业生 5,946)
                        + 在校生Excel导入 (15,975)
                        = 总计 21,940
```

**约束**: 学生的class_id必须在班级映射中存在（891条源数据无班级除外）
