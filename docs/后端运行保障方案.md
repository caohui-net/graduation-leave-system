# 后端服务运行保障方案

## 1. 进程监控（推荐：systemd）

### 创建systemd服务

```bash
sudo nano /etc/systemd/system/graduation-backend.service
```

**服务配置：**
```ini
[Unit]
Description=Graduation Leave System Backend
After=network.target

[Service]
Type=simple
User=caohui
WorkingDirectory=/home/caohui/projects/graduation-leave-system/backend
Environment="PATH=/home/caohui/projects/graduation-leave-system/backend/venv/bin"
ExecStart=/home/caohui/projects/graduation-leave-system/backend/venv/bin/python manage.py runserver 0.0.0.0:7787
Restart=always
RestartSec=10
StandardOutput=append:/tmp/backend.log
StandardError=append:/tmp/backend.log

[Install]
WantedBy=multi-user.target
```

**启用服务：**
```bash
sudo systemctl daemon-reload
sudo systemctl enable graduation-backend
sudo systemctl start graduation-backend
```

**管理命令：**
```bash
# 查看状态
sudo systemctl status graduation-backend

# 停止服务
sudo systemctl stop graduation-backend

# 重启服务
sudo systemctl restart graduation-backend

# 查看日志
sudo journalctl -u graduation-backend -f
```

**优势：**
- 自动重启（进程崩溃自动恢复）
- 系统启动时自动运行
- 统一管理接口

## 2. 简易监控脚本（无需root）

**创建监控脚本：**
```bash
nano ~/graduation-backend-monitor.sh
```

**脚本内容：**
```bash
#!/bin/bash

PROJECT_DIR="/home/caohui/projects/graduation-leave-system/backend"
VENV_PATH="$PROJECT_DIR/venv/bin/activate"
LOG_FILE="/tmp/backend.log"
PID_FILE="/tmp/backend.pid"

check_backend() {
    pgrep -f "manage.py runserver.*7787" > /dev/null
    return $?
}

start_backend() {
    cd "$PROJECT_DIR"
    source "$VENV_PATH"
    nohup python manage.py runserver 0.0.0.0:7787 >> "$LOG_FILE" 2>&1 &
    echo $! > "$PID_FILE"
    echo "[$(date)] Backend started, PID: $(cat $PID_FILE)" >> "$LOG_FILE"
}

# 检查后端是否运行
if ! check_backend; then
    echo "[$(date)] Backend not running, starting..." >> "$LOG_FILE"
    start_backend
else
    echo "[$(date)] Backend is running" >> "$LOG_FILE"
fi
```

**设置执行权限：**
```bash
chmod +x ~/graduation-backend-monitor.sh
```

**添加cron定时任务（每5分钟检查）：**
```bash
crontab -e
```

**添加行：**
```
*/5 * * * * /home/caohui/graduation-backend-monitor.sh
```

**优势：**
- 无需root权限
- 定期检查自动恢复
- 简单易维护

## 4. 日志监控

**查看实时日志：**
```bash
tail -f /tmp/backend.log
```

**查看SSO相关日志：**
```bash
grep "SSO\|callback" /tmp/backend.log | tail -20
```

**日志轮转（防止日志过大）：**
```bash
# 创建日志轮转配置
sudo nano /etc/logrotate.d/graduation-backend
```

**配置内容：**
```
/tmp/backend.log {
    daily
    rotate 7
    compress
    missingok
    notifempty
    create 0644 caohui caohui
}
```

## 推荐方案

| 方案 | 难度 | 稳定性 | 适用场景 |
|------|------|--------|---------|
| **systemd服务** | 中（需sudo） | ⭐⭐⭐⭐⭐ | 生产环境 |
| **cron监控脚本** | 低 | ⭐⭐⭐⭐ | 测试/开发环境 |
| **当前nohup** | 低 | ⭐⭐ | 临时测试 |

**生产环境建议：**
1. 使用systemd服务（自动重启）
2. 配置日志轮转（防止磁盘满）
3. 添加监控告警（进程停止时通知）

**开发环境建议：**
使用cron监控脚本（无需sudo，5分钟自动恢复）

## 快速实施（cron监控）

**一键设置：**
```bash
# 1. 创建监控脚本
cat > ~/graduation-backend-monitor.sh << 'EOF'
#!/bin/bash
PROJECT_DIR="/home/caohui/projects/graduation-leave-system/backend"
if ! pgrep -f "manage.py runserver.*7787" > /dev/null; then
    cd "$PROJECT_DIR"
    source venv/bin/activate
    nohup python manage.py runserver 0.0.0.0:7787 >> /tmp/backend.log 2>&1 &
    echo "[$(date)] Backend auto-started" >> /tmp/backend.log
fi
EOF

# 2. 设置执行权限
chmod +x ~/graduation-backend-monitor.sh

# 3. 添加cron任务（每5分钟）
(crontab -l 2>/dev/null; echo "*/5 * * * * /home/caohui/graduation-backend-monitor.sh") | crontab -

# 4. 验证
echo "✅ 监控脚本已设置，每5分钟自动检查后端运行状态"
```
