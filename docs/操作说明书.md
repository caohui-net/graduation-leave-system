**版本：** v0.1  
**最后更新：** 2026-06-02  
**适用范围：** 系统管理员、学生、审批人员

---

## 1. 系统概述

### 1.1 系统简介

毕业生离校申请审批系统用于管理毕业生离校申请的提交、审批和备案流程。系统采用两级审批模式：宿管员审批 → 辅导员审批，学工部进行备案查询。

### 1.2 用户角色

| 角色 | 权限 | 主要功能 |
|------|------|---------|
| 学生 (student) | 提交申请 | 填写离校信息、上传附件、查看申请状态 |
| 宿管员 (dorm_manager) | 第一级审批 | 审批宿舍清退相关事项 |
| 辅导员 (counselor) | 第二级审批 | 审批学业和档案相关事项 |
| 学工部 (dean) | 备案查询 | 查看已通过申请，用于存档和统计 |

### 1.3 审批流程

```
学生提交申请
    ↓
宿管员审批（通过/驳回）
    ↓
辅导员审批（通过/驳回）
    ↓
状态变为"已通过"
    ↓
学工部备案查询
```

## 2. 系统部署（管理员）

### 2.1 环境要求

**后端：**
- Python 3.8+
- PostgreSQL 12+
- Redis 6+

**前端：**
- 微信开发者工具
- Node.js 14+（用于demo-web）

### 2.2 后端部署

```bash
# 1. 克隆代码
git clone <repository-url>
cd graduation-leave-system/backend

# 2. 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements/dev.txt  # 开发环境
# pip install -r requirements/prod.txt  # 生产环境

# 4. 配置数据库
# 编辑 config/settings/dev.py 或设置环境变量
# DATABASE_URL=postgresql://user:pass@localhost/dbname

python manage.py migrate

# 6. 导入测试数据（可选）
python manage.py seed_data

# 7. 启动开发服务器
python manage.py runserver 0.0.0.0:8001

# 8. 配置小程序后端地址
# 编辑 miniprogram/services/api.ts
# baseUrl: 'http://localhost:8001'
```

## 3. 用户操作指南

### 3.1 学生操作

#### 3.1.1 登录系统

**微信小程序：**
1. 打开"毕业离校申请"小程序
2. 输入学号和密码
3. 点击"登录"

**Demo Web：**
1. 访问系统URL
2. 选择"学生申请"页面
3. 输入学号：2020001，密码：password123

#### 3.1.2 提交离校申请

1. 登录后自动进入"学生申请"页面
2. 填写离校原因（必填，最多500字）
3. 选择离校日期（必须为今天或之后）
4. 点击"提交申请"
5. 提交成功后自动跳转到申请详情页
6. 在详情页可上传附件（可选）
   - 支持格式：JPG, PNG, PDF, DOC, DOCX
   - 单文件最大10MB
   - 可上传多个文件
   - 可随时删除已上传的附件

#### 3.1.3 查看申请状态

1. 在"我的申请"页面查看申请列表
2. 点击申请卡片进入详情页
3. 查看当前审批状态：
   - **待宿管员审批**：等待宿管员处理
   - **待辅导员审批**：宿管员已通过，等待辅导员处理
   - **已通过**：所有审批通过，可以办理离校
   - **已驳回**：审批未通过，查看驳回原因

#### 3.1.4 重新提交被驳回的申请

1. 查看被驳回申请的详情
2. 阅读驳回原因
3. 修改申请内容后重新提交
4. 审批流程从第一级（宿管员）重新开始

### 3.2 宿管员操作

#### 3.2.1 登录系统

⚠️ **注意：** 当前版本宿管员功能尚未完全实现：
- 种子数据未包含宿管员测试账号
- 微信小程序审批页面仅支持辅导员和学工部角色访问

**Demo Web：**
宿管员功能在Demo Web中可正常使用（需先配置测试账号）

#### 3.2.2 查看待审批申请

1. 在"审批列表"页面选择"待审批" Tab
2. 查看当前待处理的申请列表
3. 显示信息：
   - 申请ID
   - 学生信息
   - 提交时间
   - 当前状态

#### 3.2.3 审批申请

1. 点击待审批申请进入详情页
2. 查看申请信息：
   - 学生基本信息
   - 离校原因
   - 离校日期
   - 附件列表（如有）
3. 填写审批意见（可选，最多200字）
4. 点击操作按钮：
   - **通过**：同意申请，流转到辅导员
   - **拒绝**：驳回申请，学生可重新提交

#### 3.2.4 查看审批历史

1. 在"审批列表"切换Tab筛选：
   - **待审批**：显示 decision=pending 的审批记录（默认）
   - **已审批**：显示 decision=approved 的审批记录
   - **全部**：显示所有审批记录（包括待审批、已通过、已拒绝）
2. 点击任意审批记录查看详情和审批意见

**注意：** 小程序当前不支持单独查看"已拒绝"审批，请使用"全部"Tab查看所有记录。

### 3.3 辅导员操作

#### 3.3.1 登录系统

**微信小程序/Demo Web：**
1. 输入账号：counselor，密码：password123
2. 点击"登录"
3. 自动进入"审批列表"页面

#### 3.3.2 审批流程

操作步骤与宿管员相同：
1. 查看待审批列表（仅显示宿管员已通过的申请）
2. 点击申请查看详情
3. 查看审批记录（可见宿管员的审批意见）
4. 填写审批意见
5. 点击"通过"或"拒绝"

**注意：** 辅导员审批通过后，申请状态变为"已通过"，审批流程结束。

### 3.4 学工部操作

#### 3.4.1 登录系统

**微信小程序/Demo Web：**
1. 输入账号：dean，密码：password123
2. 点击"登录"
3. 自动进入"备案查询"页面

#### 3.4.2 查询已通过申请

1. 在"备案查询"页面查看所有已通过的申请

4. **注意：** 学工部角色为只读权限，无审批按钮

1. 点击申请进入详情页
2. 查看信息：

   - 申请内容

## 4. 常见问题

### 4.1 登录问题

**Q: 忘记密码怎么办？**  
A: 联系系统管理员重置密码，或使用管理后台重置。

**Q: 登录后页面显示"权限不足"？**  
A: 检查账号角色是否正确，确认是否有相应权限。

### 4.2 申请提交问题

**Q: 提交时提示"离校日期无效"？**  
A: 离校日期必须选择今天或之后的日期，不能选择过去的日期。

**Q: 提示"已有待审批或已通过的申请"？**  
A: 系统不允许同时存在多个待审批或已通过的申请。如需修改，请等待当前申请被驳回，或联系管理员撤销当前申请。

**Q: 提示"宿舍清退未完成"？**  
A: 需先完成宿舍清退流程。清退完成后系统会自动解除限制，方可提交离校申请。

**Q: 提示"班级映射不存在"？**  
A: 您所在班级尚未配置审批流程（辅导员或宿管员信息缺失）。请联系系统管理员配置班级映射关系。

**Q: 附件上传失败？**  
A: 检查：
- 文件格式是否支持（JPG/PNG/PDF/DOC/DOCX）
- 文件大小是否超过10MB
- 网络连接是否正常

**Q: 提交后能修改吗？**  
A: 提交后无法直接修改。如需修改，可联系审批人驳回申请后重新提交。

### 4.3 审批问题

**Q: 待审批列表为空？**  
A: 可能原因：
- 当前没有待处理的申请
- 申请已被其他审批人处理
- 角色权限配置错误

**Q: 审批后能撤销吗？**  
A: 审批操作不可撤销。如需修改，只能驳回申请让学生重新提交。

**Q: 为什么看不到审批按钮？**  
A: 学工部角色为只读权限，只能查询已通过申请，无审批权限。

### 4.4 系统性能

**Q: 页面加载慢？**  
A: 检查：
- 后端服务是否正常运行
- 数据库连接是否正常
- 网络延迟情况

## 附录A：API端点说明

**认证：**
- POST /api/auth/login - 用户登录

**申请管理：**
- POST /api/applications/ - 创建申请
- GET /api/applications/ - 查询申请列表
- GET /api/applications/{id}/ - 查询申请详情

**审批管理：**
- GET /api/approvals/ - 查询审批列表（支持decision参数：pending/approved/rejected/all）
- POST /api/approvals/{id}/approve/ - 通过审批
- POST /api/approvals/{id}/reject/ - 驳回审批

**附件管理：**
- GET /api/applications/{id}/attachments/ - 获取附件列表
- POST /api/applications/{id}/attachments/ - 上传附件
- GET /api/attachments/{id}/download/ - 下载附件
- DELETE /api/attachments/{id}/ - 删除附件

详细API文档参见：`docs/api/contract-v0.3.md`
