# UI优化需求 - 配色调整、响应式设计与附件上传

**文档编号：** 05-ui-enhancement-color-responsive  
**创建时间：** 2026-06-02  
**讨论主题：** 基于参考网站调整配色 + PC/移动端响应式适配 + 申请表单附件上传UI

---

## 1. 背景

当前UI demo已完成基础实施（蓝色主题#1890FF，移动端375px固定宽度）。用户提出三个新需求：

1. **配色调整：** 参考 https://xuegong.hgnu.edu.cn/main.htm 的风格进行UI配色
2. **响应式设计：** 支持PC端与移动端自适应布局
3. **附件上传UI：** 在申请表单中添加附件上传功能界面

---

## 2. 需求详情

### 需求1：参考网站配色方案

**参考网站：** https://xuegong.hgnu.edu.cn/main.htm

**任务：**
- 分析参考网站的主色调、辅助色、背景色、文本色
- 提取完整配色方案（Primary, Secondary, Accent, Background, Text）
- 将配色应用到当前UI系统（替换现有#1890FF蓝色主题）
- 保持设计一致性和可读性

**输出：**
- 完整色板定义（Hex色值 + 用途说明）
- CSS变量更新方案
- 视觉对比示例（当前配色 vs 新配色）

### 需求2：响应式设计（PC + 移动端）

**当前状态：**
- 固定宽度：375px（移动端）
- 仅支持手机端查看
- 布局不适配大屏

**目标：**
- **移动端（<768px）：** 单列布局，保持当前设计
- **平板端（768px-1024px）：** 适度扩展，优化间距
- **桌面端（>1024px）：** 双列或居中布局，最大宽度限制

**关键点：**
- Tab导航在PC端改为侧边栏或顶部横向导航
- 卡片布局在PC端支持多列网格
- 表单在PC端宽度限制（不要过宽）
- 时间轴在PC端保持可读性

**输出：**
- 响应式断点策略（breakpoints）
- 布局调整方案（移动优先 vs 桌面优先）
- CSS媒体查询实现
- 桌面端导航结构设计

### 需求3：申请表单附件上传UI

**当前状态：**
- 申请详情页（detail）已有附件管理功能（查看/下载/删除）
- 学生申请页（student-application）无附件上传入口
- 申请提交后无法上传初始附件

**目标：**
- 在学生申请表单中添加附件上传UI
- 支持多文件上传（图片、PDF、Word文档）
- 实时显示已选文件列表
- 文件类型和大小校验（<10MB，支持.jpg/.png/.pdf/.doc/.docx）
- 附件类型选择（宿舍清退/图书馆清书/财务结清/其他）

**关键点：**
- 上传时机：提交申请前预览，提交时一并发送
- 移动端：微信小程序 `wx.chooseMessageFile` API
- PC端：HTML5 file input + drag-and-drop
- UI反馈：上传进度、成功/失败状态
- 已有后端API：`/api/attachments/upload/` (multipart/form-data)

**输出：**
- 附件上传UI组件设计（移动端+PC端）
- 文件选择交互流程
- 附件列表展示方案
- 错误处理和用户反馈

## 4. 响应式设计方案（待讨论）

### 4.1 断点策略（Breakpoints）

**提议：**
```css
/* 移动端优先 */
/* Base: <768px (mobile) */
@media (min-width: 768px) { /* Tablet */ }
@media (min-width: 1024px) { /* Desktop */ }
@media (min-width: 1440px) { /* Large Desktop */ }
```

### 4.2 布局调整

| 组件 | 移动端 (<768px) | 桌面端 (>1024px) |
|-----|----------------|-----------------|
| 容器宽度 | 100% | max-width: 1200px, 居中 |
| Tab导航 | 顶部横向3列 | 侧边栏或顶部横向 |
| 卡片列表 | 单列 | 2-3列网格 |
| 表单 | 单列 | max-width: 600px |
| 详情页 | 单列 | 左侧详情+右侧操作 |

### 4.3 实施优先级

**Phase 1（必须）：**
- 提取参考网站配色
- 应用新配色到现有demo
- 基础响应式（容器max-width + 媒体查询）
- 附件上传UI基础功能（移动端）

**Phase 2（建议）：**
- Tab导航桌面端优化
- 卡片列表网格布局
- 表单宽度限制
- 附件上传PC端优化（拖拽上传）

**Phase 3（可选）：**
- 详情页双列布局
- 高级交互优化（hover效果、动画）
- 附件预览功能增强

## 8. Gemini 分析结果及建议

Gemini 已完成对参考网站 `https://xuegong.hgnu.edu.cn/main.htm` 的分析并总结了配色、响应式设计以及附件上传的交互流程。详细内容已记录在：`docs/ui-design-proposals/01-analysis-and-responsive-plan.md`

### 简要总结：

**1. 配色方案**
- **主色调 (Primary)**：`#971010` (深红/绛红)，展现学术与正式感。
- **辅助色 (Secondary)**：`#B56D18` (土黄/金棕)、`#d99749` (暖金)。
- **背景/边框**：浅灰 (`#f2f2f2`) 及暖米色 (`#f7f3eb`) 为主。
- **整体风格**：正式、学术、稳重。

**2. 响应式策略**
- 移动端优先 (`max-width: 100%`, 单列布局)。
- 桌面端 (`>= 1024px`) 采用居中布局，最大宽度限制为 `1004px` 或 `1200px`，表单进行限宽处理，列表转为多列。

**3. 附件上传UI**
- 移动端：复用 `wx.chooseMessageFile`，暂存到表单内，提交时统一上传。
- PC端：加入拖拽区 (Drag-and-Drop) 及 `<input type="file" multiple>`。
- 文件展示为紧凑列表项，附带文件名、大小和删除按钮。
