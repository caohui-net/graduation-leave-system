## 2. 系统完整功能清单

根据 `docs/PROJECT-SUMMARY.md` 和 `docs/design/2026-05-27-system-design.md`：

### 2.1 核心功能模块
1. **用户认证**
   - 学号+密码登录
   - 微信OAuth2登录
   - 首次登录强制修改密码
   - 忘记密码（如需要）

2. **学生功能**
   - 提交离校申请（填写原因、选择日期）
   - 上传附件（宿舍清退证明、图书馆清书证明、财务结清证明、其他）
   - 查看申请状态（草稿、待审批、已通过、已拒绝）
   - 查看审批历史
   - 查看审批意见
   - 申请被拒后重新提交

3. **辅导员功能**
   - 查看待审批列表（本班级学生）
   - 审批申请（通过/拒绝 + 意见）
   - 查看已审批历史
   - 过滤筛选（按状态、按日期）

4. **院长功能**
   - 查看待审批列表（本学院学生）
   - 审批申请（通过/拒绝 + 意见）
   - 查看已审批历史
   - 过滤筛选

5. **通知功能**
   - 申请提交成功通知
   - 审批通过通知
   - 审批拒绝通知
   - 审批超时提醒（给审批人）
   - 宿舍清退阻塞通知

6. **个人中心**
   - 查看个人信息（姓名、学号、角色、学院、专业、班级）
   - 修改密码
   - 退出登录

7. **错误与异常**
   - 权限不足（403）
   - 资源不存在（404）
   - 网络错误
   - 服务器错误（500）
   - 宿舍清退阻塞提示

## 4. UI覆盖缺口分析

### 4.1 缺失页面

| 功能模块 | 缺失页面 | 优先级 | 说明 |
|---------|---------|--------|------|
| 首次登录 | 强制修改密码页 | P1 | 安全要求（初始密码=学号） |
| 个人中心 | 个人中心页 | P1 | 查看信息、修改密码、退出 |
| 学生功能 | 我的申请列表页 | P2 | 学生查看历史申请（可选，可复用approvals页） |
| 通知功能 | 通知列表页 | P2 | 查看历史通知（可选） |
| 错误处理 | 权限不足页（403） | P2 | 角色权限错误提示 |
| 错误处理 | 页面不存在（404） | P3 | 资源不存在提示 |
| 错误处理 | 网络错误页 | P3 | 统一网络异常处理 |

### 4.2 功能缺口（现有页面）

#### 学生申请页 (`student-application`)
- ✅ 基础表单（原因、日期）
- ❌ 附件上传入口（当前无）
- ❌ 草稿保存功能（当前无）
- ❌ 宿舍清退阻塞提示（当前有，需优化视觉）

#### 审批列表页 (`approvals`)
- ✅ Tab切换（全部/待审批/已审批）
- ❌ 高级筛选（按日期、按学生）
- ❌ 批量操作（如需要）
- ❌ 空状态视觉优化

#### 申请详情页 (`detail`)
- ✅ 基本信息展示
- ✅ 审批记录时间轴
- ✅ 附件管理（上传/下载/删除）
- ✅ 审批操作（通过/拒绝）
- ❌ 审批意见输入（当前直接通过/拒绝，无意见输入框）
- ❌ 拒绝后重新提交入口（学生角色）

#### 登录页 (`login`)
- ✅ 学号+密码登录
- ❌ 微信OAuth2登录（当前无）
- ❌ 忘记密码入口（如需要）
- ❌ 首次登录引导

## 6. 导航结构提议

### 6.1 底部Tab导航（tabBar）

**方案A：角色统一Tab（推荐）**
```
学生角色：[我的申请] [个人中心]
辅导员/院长角色：[待审批] [个人中心]
```

**方案B：固定Tab**
```
全角色：[首页] [审批] [个人中心]
- 首页：学生显示"我的申请"，审批人显示"待审批"
```

### 6.2 页面跳转关系

```
登录页 ──登录成功──> 首页（根据角色）
                      │
        ┌─────────────┴─────────────┐
        │                           │
    学生角色                    审批人角色
        │                           │
   [我的申请]                   [待审批列表]
        │                           │
    点击申请                     点击申请
        │                           │
        └──────> [申请详情页] <──────┘
                      │
            ┌─────────┴─────────┐
            │                   │
        [附件管理]           [审批操作]
                              (通过/拒绝+意见)

[个人中心] ──> [修改密码页]
            └──> 退出登录 ──> [登录页]
```

## 10. 向用户确认的问题

1. **导航结构：** 是否需要底部tabBar？方案A（角色动态）还是方案B（固定Tab）？
2. **优先级：** Phase 1（修改密码、个人中心）是否必须？还是先完成Phase 0？
3. **附件上传：** 申请页是否需要附件上传入口？还是仅在详情页上传？
4. **通知页面：** 是否需要独立通知列表页？还是用Toast/Banner提示？
5. **微信登录：** 是否需要微信OAuth2登录UI？还是仅支持学号+密码？

---

**文档总结：** 当前UI讨论范围过窄（3页面），系统实际需要6-10个页面。建议分阶段实施：Phase 0完善现有4页面（2-3h），Phase 1新增核心页面（3-4h），Phase 2-3可选增强（3-5h）。需用户确认优先级和导航结构。
