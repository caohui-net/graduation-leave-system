## 五、对接流程说明

### 5.1 管理端登录流程

```
1. 管理员在青橄榄一站式管理平台点击"毕业离校系统"

2. 青橄榄生成authorization token

3. 青橄榄跳转到：
   http://218.75.196.218:7788/admin/sso-callback?authorization=bearer%20{token}

4. 本系统前端提取token参数

5. 本系统前端调用：
   POST http://218.75.196.218:7787/api/sso/qingganlian/admin/login
   Body: {"authorization": "bearer {token}"}

6. 本系统后端调用青橄榄API验证token：
   POST https://zhhq.huanghuai.edu.cn/api/open-api/auth/verify-user
   Headers: {appKey, timestamp, randStr, sign, encryptionType}
   Body: {"authorization": "bearer {token}"}

7. 青橄榄返回用户信息（username, name, phone, tenant_code等）

8. 本系统创建本地session，生成JWT token

9. 本系统返回JWT给前端

10. 用户登录成功，前端存储JWT用于后续请求
```

## 六、技术实现说明

### 6.1 签名算法

严格遵循青橄榄签名规范：
1. 将 `app_secret`, `timestamp`, `rand_str` 排序
2. 拼接为字符串
3. SHA1加密生成签名

### 6.2 请求Header

所有调用青橄榄API的请求包含：
- `appKey`: cb6f276a42042179e90cd79c4126e075
- `timestamp`: Unix时间戳
- `randStr`: 16位随机字符串
- `sign`: 认证签名
- `encryptionType`: sha1

### 6.3 超时与重试

- 请求超时：30秒
- 失败重试：3次
- 降级策略：API失败时引导用户刷新页面重试
