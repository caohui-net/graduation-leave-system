## 截图获取步骤

### 方案1：微信小程序截图（推荐）

**前置条件：**
- 微信开发者工具已安装
- 小程序代码已导入（`miniprogram/`目录）
- 后端API服务运行中（docker-compose up）

**操作步骤：**

1. 启动后端服务
```bash
docker compose up -d
```

2. 打开微信开发者工具
   - 导入项目：选择`miniprogram/`目录
   - 配置appid（测试号）
   - 点击"编译"

3. 按角色截图

**学生角色（2020001）：**
- 登录界面
- 提交申请界面
- 申请列表界面
- 申请详情界面
- 审批记录详情

**宿管员角色（M001）：**
- 登录界面
- 待审批列表
- 审批界面
- 多宿管员提示
- 审批记录

**辅导员角色（T001）：**
- 登录界面
- 待审批列表
- 审批界面
- 审批记录

4. 截图保存到 `docs/screenshots/` 目录
   - 命名规范：`{角色}-{功能}.png`
   - 例如：`student-login.png`, `manager-approve.png`

5. 插入截图到DOCX
   - 打开 `用户操作手册.docx`
   - 找到红色斜体的截图占位符
   - 删除占位符文本
   - 插入对应截图（图片路径已标注）

### 方案2：API测试截图（备选）

如果小程序无法运行，可使用Postman/Apifox等API测试工具：

1. 导入API端点（参考`SYSTEM-OPERATIONS-GUIDE.md`）
2. 测试完整审批流程
3. 截取请求/响应示例
4. 配合流程图说明操作逻辑

**缺点：** 无法展示真实UI，仅能演示后端交互

## 截图规范

**尺寸：** 宽度统一1200px（缩放至文档宽度）  
**格式：** PNG（保留清晰度）  
**内容：** 避免包含真实敏感信息（测试账号可见）  
**标注：** 关键按钮/输入框可添加红框标注
