## 一、系统概述

### 1.1 系统目的

本系统旨在实现毕业生离校申请的在线化、规范化管理，替代传统纸质审批流程，提高审批效率，减少人工成本。

### 1.2 主要功能

- **学生端：** 在线提交离校申请、上传证明材料、查看审批进度、接收审批通知
- **辅导员端：** 审批本年级学生的离校申请、查看审批历史、接收待办提醒
- **学工部端：** 审批所有学生的离校申请、查看全校审批统计、导出审批数据
- **系统管理：** 用户管理、权限配置、系统参数设置

### 1.3 使用平台

- **移动端：** iOS/Android App（React Native开发）
- **微信小程序：** 微信内使用，无需安装
- **管理后台：** Web浏览器访问（可选）

## 二、业务流程

### 2.1 申请提交流程

```
学生登录 → 填写申请信息 → 上传证明材料 → 提交申请 → 等待审批
```

**必填信息：**
- 学号、姓名、专业、班级
- 计划离校日期
- 联系电话、家庭地址

**必传材料：**
- 宿舍清退证明（照片或PDF）
- 图书馆清书证明（照片或PDF）
- 财务结清截图（照片或PDF）

### 2.2 审批流程

```
学生提交 → 辅导员审批 → 学工部审批 → 审批完成
           ↓（驳回）      ↓（驳回）
         学生修改 ←──────┘
```

**审批时限：**
- 辅导员审批：提交后1个工作日内
- 学工部审批：辅导员通过后1个工作日内
- 超时提醒：系统自动发送微信通知

**审批结果：**
- **通过：** 进入下一审批环节或完成审批
- **驳回：** 学生收到通知，修改后重新提交

### 2.3 通知机制

**微信通知场景：**
- 学生提交申请 → 通知辅导员
- 辅导员审批通过 → 通知学生和学工部
- 辅导员驳回 → 通知学生
- 学工部审批通过 → 通知学生（审批完成）
- 学工部驳回 → 通知学生
- 审批超时 → 通知审批人

## 三、系统特色

### 3.1 双通道认证

**学号密码登录：**
- 适用于首次使用或更换设备
- 学号由学校统一导入
- 初始密码为学号，首次登录强制修改密码
- 密码要求：至少8位，包含字母和数字

**微信授权登录：**
- 绑定后一键登录，无需输入密码
- 支持微信消息推送
- 更换手机后需重新绑定

### 3.2 外部系统对接

**宿舍管理系统对接：**
- 学生提交申请时，系统自动验证宿舍是否已清退
- 验证失败时，提示学生先完成宿舍清退
- 如外部系统不可用，允许手动上传证明

**对接方式：**
- API接口对接（推荐）
- 手动上传证明（降级方案）

### 3.3 数据安全

**传输安全：**
- 全站HTTPS加密传输
- API接口JWT Token认证

**存储安全：**
- 密码加密存储（bcrypt算法）
- 附件文件权限控制（只能查看自己的）
- 审计日志记录（所有操作可追溯）

**隐私保护：**
- 学生只能查看自己的申请
- 辅导员只能查看负责班级的学生
- 学工部可以查看所有学生

## 四、系统角色与权限

### 4.1 角色定义

| 角色 | 权限范围 | 主要功能 |
|------|----------|----------|
| **学生** | 本人申请 | 提交申请、查看进度、上传附件 |
| **辅导员** | 本年级学生 | 审批申请、查看历史、接收通知 |
| **学工部** | 全校学生 | 审批申请、查看统计、导出数据 |
| **系统管理员** | 全部功能 | 用户管理、权限配置、系统设置 |

### 4.2 权限说明

**学生权限：**
- ✓ 创建离校申请
- ✓ 修改草稿状态的申请
- ✓ 提交申请
- ✓ 查看自己的申请列表和详情
- ✓ 上传附件（最多5个，每个最大10MB）
- ✓ 查看审批历史
- ✗ 查看其他学生的申请

**辅导员权限：**
- ✓ 查看负责班级学生的申请列表
- ✓ 审批负责班级学生的申请（通过/驳回）
- ✓ 填写审批意见
- ✓ 查看审批历史
- ✓ 接收待审批通知
- ✗ 审批其他班级学生

**学工部权限：**
- ✓ 查看所有学生的申请列表
- ✓ 审批所有学生的申请（通过/驳回）
- ✓ 填写审批意见
- ✓ 查看全校审批统计
- ✓ 导出审批数据（Excel）
- ✓ 查看审批历史

## 五、系统部署

### 5.1 部署方式

**本地部署（推荐）：**
- 部署在学校内网服务器
- 数据存储在学校自有数据库
- 完全掌控数据安全

**部署要求：**
- 服务器：4核CPU、8GB内存、100GB硬盘
- 操作系统：Linux（Ubuntu 20.04+或CentOS 7+）
- 数据库：PostgreSQL 12+
- 容器：Docker 20.10+、Docker Compose 2.0+

### 5.2 系统容量

**设计容量（单实例）：**
- 在线用户：500人
- 并发请求：50个
- 峰值QPS：100次/秒
- 响应时间：<200ms（95%请求）

**数据容量：**
- 学生用户：10,000人
- 年度申请：5,000份
- 附件存储：50GB/年

### 5.3 备份策略

**数据库备份：**
- 每日全量备份（凌晨2点）
- 保留最近30天备份
- 备份文件加密存储

**附件备份：**
- 每周全量备份（周日凌晨）
- 保留最近12周备份

## 六、常见问题

### 6.1 学生常见问题

**Q1：忘记密码怎么办？**
A：联系辅导员或系统管理员重置密码。

**Q2：微信绑定失败怎么办？**
A：确认学号和密码正确，如仍失败请联系管理员。

**Q3：上传附件失败怎么办？**
A：检查文件大小（<10MB）和格式（jpg/png/pdf/doc/docx），如仍失败请联系管理员。

**Q4：申请被驳回后如何修改？**
A：在申请详情页点击"修改"按钮，修改后重新提交。

**Q5：审批进度在哪里查看？**
A：登录后在"我的申请"列表中查看，点击详情可查看审批历史。

### 6.2 审批人常见问题

**Q1：如何查看待审批申请？**
A：登录后在"待审批"列表中查看，点击进入审批页面。

**Q2：审批意见是否必填？**
A：驳回时必须填写驳回原因，通过时可选填。

**Q3：审批后能否撤销？**
A：审批后不能撤销，如需修改请联系系统管理员。

**Q4：如何导出审批数据？**
A：学工部用户在"审批统计"页面点击"导出Excel"按钮。

### 6.3 系统管理问题

**Q1：如何批量导入学生用户？**
A：准备Excel模板（学号、姓名、专业、班级、辅导员），在管理后台上传。

**Q2：如何修改审批时限？**
A：在系统配置页面修改"审批时限"参数（单位：小时）。

**Q3：如何查看系统日志？**
A：在管理后台"审计日志"页面查看，支持按用户、操作、时间筛选。

**Q4：如何对接宿舍管理系统？**
A：参考《数据对接说明文档》，提供API接口或数据导出。
