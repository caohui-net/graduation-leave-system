# Staging环境测试清单

**执行时间**: 部署到staging后
**执行人**: 开发/测试团队
**目标**: 验证新功能不影响现有功能，留校申请流程正常

---

## 1. 回归测试 - 原离校流程

### 测试用例1.1: 提交离校申请
- [ ] 登录学生账号
- [ ] 提交离校申请（不指定application_type，默认为leave_school）
- [ ] 验证：宿管员收到审批通知
- [ ] 宿管员审批通过
- [ ] 验证：辅导员收到审批通知
- [ ] 辅导员审批通过
- [ ] 验证：申请状态变为已批准

**预期**: 流程不受影响，审批路径正常

### 测试用例1.2: 管理员查看申请列表
- [ ] 登录管理员账号
- [ ] 查看申请列表
- [ ] 验证：能看到所有离校申请
- [ ] 验证：application_type字段显示正确

---

## 2. 功能测试 - 留校申请（Feature Flag=false）

### 测试用例2.1: Feature Flag关闭时提交留校申请
- [ ] 设置环境变量 ENABLE_STAY_SCHOOL=false
- [ ] 重启服务
- [ ] 尝试提交application_type=stay_school的申请
- [ ] 验证：返回403错误，提示功能未开放

**预期**: Feature Flag生效，留校功能被阻止

---

## 3. 功能测试 - 留校申请（Feature Flag=true）

### 测试用例3.1: 提交留校申请
- [ ] 设置环境变量 ENABLE_STAY_SCHOOL=true
- [ ] 重启服务
- [ ] 登录学生账号
- [ ] 提交留校申请：
  - application_type: stay_school
  - stay_start_date: 2026-07-01
  - stay_end_date: 2026-08-31
  - stay_reason: exam_prep
  - contact_phone: 13800138000
- [ ] 验证：申请创建成功
- [ ] 验证：申请状态为PENDING_COUNSELOR（跳过宿管员）
- [ ] 验证：辅导员收到审批通知

**预期**: 留校申请直接进入辅导员审批

### 测试用例3.2: 留校申请审批流程
- [ ] 登录辅导员账号
- [ ] 查看待审批列表
- [ ] 验证：能看到留校申请
- [ ] 审批通过
- [ ] 验证：申请状态变为已批准
- [ ] 验证：学生收到通过通知

**预期**: 一级审批流程正常

### 测试用例3.3: 留校申请字段验证
- [ ] 尝试提交缺少stay_start_date的留校申请
- [ ] 验证：返回400错误，提示缺少必填字段
- [ ] 尝试提交缺少stay_reason的留校申请
- [ ] 验证：返回400错误

**预期**: 字段验证生效
