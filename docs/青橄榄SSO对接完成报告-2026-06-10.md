## Token自动恢复功能

### 实现原理
```javascript
// 页面加载时从localStorage恢复token
function restoreAuthState() {
    const savedToken = localStorage.getItem('auth_token');
    const savedUser = localStorage.getItem('user_info');
    if (savedToken) {
        currentToken = savedToken;
        currentUser = JSON.parse(savedUser);
    }
}

// DOM加载完成后初始化UI
document.addEventListener('DOMContentLoaded', function() {
    if (currentToken && currentUser) {
        initializeUIAfterRestore(currentUser);
    }
});
```

### 功能特性
- ✅ Token持久化存储（localStorage）
- ✅ 页面刷新保持登录状态
- ✅ Token有效期内（7天）免登录
- ✅ 自动初始化UI（隐藏登录框，显示业务界面）
- ✅ 管理端和移动端共用机制

## 技术实现

### 后端改造

**简化流程：**
- 原设计：OAuth token exchange（2次API调用）
- 新实现：直接创建用户（0次API调用）
- 原因：青橄榄平台实际采用简化流程，直接返回JWT + 用户标识

**代码变更：**
```python
# backend/apps/sso_qingganlian/views.py
@api_view(['POST'])
@permission_classes([AllowAny])
def admin_login(request):
    authorization = request.data.get('authorization')
    username = request.data.get('username')
    
    # 直接创建User，无需API调用
    with transaction.atomic():
        user, created = User.objects.select_for_update().get_or_create(
            user_id=username,
            defaults={'name': username, 'role': 'admin', ...}
        )
    
    # 返回本系统JWT token
    refresh = RefreshToken.for_user(user)
    return Response({'token': str(refresh.access_token), ...})
```

### 前端增强

**回调页面：**
```javascript
// 提取回调参数（兼容大小写）
const params = new URLSearchParams(window.location.search);
const authorization = params.get('authorization') || params.get('Authorization');

// 调用后端API

// 保存token并跳转
```
