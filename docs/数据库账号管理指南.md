# 数据库账号管理指南

**版本**: v1.0  
**适用环境**: 生产环境  
**最后更新**: 2026-06-27

---

## 账号体系

### 1. 应用账号 (`app_prod_user`)
- **用途**: 应用运行时使用
- **权限**: 仅DML（SELECT/INSERT/UPDATE/DELETE）
- **禁止**: DDL操作（CREATE/DROP/ALTER）
- **配置位置**: `.env.production` 的 `DB_USER`

### 2. 迁移账号 (`migration_user`)
- **用途**: 执行数据库迁移（发布时）
- **权限**: 完整DDL+DML权限
- **使用时机**: 仅发布流程中使用
- **配置位置**: 发布脚本中临时设置

### 3. 只读账号 (`readonly_user`)
- **用途**: 监控、报表、数据分析
- **权限**: 仅SELECT
- **使用者**: 运维、数据分析人员

## 初始化步骤

### 1. 执行SQL脚本
```bash
psql -h prod-db -U postgres -d graduation_prod -f backend/scripts/setup-prod-db-accounts.sql
```

### 2. 修改默认密码
```sql
ALTER USER app_prod_user PASSWORD 'your_strong_password_here';
ALTER USER migration_user PASSWORD 'your_strong_password_here';
ALTER USER readonly_user PASSWORD 'your_strong_password_here';
```

### 3. 更新应用配置
```bash
# .env.production
DB_USER=app_prod_user
DB_PASSWORD=your_strong_password_here
```

### 4. 更新发布脚本
```bash
# scripts/deploy-production.sh
export MIGRATION_DB_USER=migration_user
export MIGRATION_DB_PASSWORD=migration_password
python manage.py migrate --database=default
```

## 权限验证

### 验证应用账号
```sql
-- 应返回 false（无DDL权限）
SELECT has_table_privilege('app_prod_user', 'users', 'CREATE');

-- 应返回 true（有DML权限）
SELECT has_table_privilege('app_prod_user', 'users', 'SELECT');
SELECT has_table_privilege('app_prod_user', 'users', 'INSERT');
```

### 验证迁移账号
```sql
-- 应返回 true（有DDL权限）
SELECT has_table_privilege('migration_user', 'users', 'CREATE');
```

### 验证只读账号
```sql
-- 应返回 false（无写权限）
SELECT has_table_privilege('readonly_user', 'users', 'INSERT');

-- 应返回 true（有读权限）
SELECT has_table_privilege('readonly_user', 'users', 'SELECT');
```

## 发布流程集成

### 发布前
1. 使用 `app_prod_user` 备份数据库
2. 验证 `migration_user` 密码可用

### 发布时
1. 临时切换为 `migration_user`
2. 执行 `python manage.py migrate`
3. 验证迁移结果
4. 切换回 `app_prod_user`

### 发布后
1. 验证应用使用 `app_prod_user` 连接
2. 检查数据库连接数正常

## 安全最佳实践

### 密码管理
- 使用强密码（16+字符，包含大小写/数字/特殊字符）
- 定期轮换密码（每90天）
- 不在代码中硬编码密码
- 使用环境变量或密钥管理工具

### 连接限制
```sql
-- 限制并发连接数
ALTER USER app_prod_user CONNECTION LIMIT 50;
ALTER USER migration_user CONNECTION LIMIT 2;
ALTER USER readonly_user CONNECTION LIMIT 10;
```

### 审计日志
```sql
-- 启用连接日志
ALTER DATABASE graduation_prod SET log_connections = on;
ALTER DATABASE graduation_prod SET log_disconnections = on;
```

## 故障排查

### 问题1: 应用无法连接
```bash
# 检查账号是否锁定
SELECT rolname, rolcanlogin FROM pg_roles WHERE rolname = 'app_prod_user';

# 检查连接数限制
SELECT count(*) FROM pg_stat_activity WHERE usename = 'app_prod_user';
```

### 问题2: 迁移失败（权限不足）
```bash
# 验证迁移账号权限
psql -U migration_user -d graduation_prod -c "\dp users"
```

### 问题3: 应用尝试执行DDL
```bash
# 查看错误日志
tail -f /var/log/postgresql/postgresql.log | grep "permission denied"

# 应该阻止，这说明代码存在问题
```

## 回滚方案

### 临时恢复完整权限（紧急情况）
```sql
-- 仅用于紧急修复，完成后立即撤销
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO app_prod_user;

-- 修复完成后撤销
REVOKE CREATE, DROP, ALTER ON ALL TABLES IN SCHEMA public FROM app_prod_user;
```
