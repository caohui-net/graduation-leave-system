# 数据补充请求：271名研究生学号信息

**日期：** 2026-06-06  
**紧急程度：** 中等  
**影响范围：** 271名研究生

---

## 问题说明

在数据合并过程中，发现271名研究生的学号字段为空，系统已为他们生成临时ID（格式：TMP2026_XXXX），但临时ID不能替代官方学号用于正式记录和系统对接。

### 数据特征

- **人数：** 271名
- **学生类型：** 全部为研究生
- **年级：** 主要为2023级
- **数据完整性：** 有姓名、学院、专业、楼栋、寝室号，**仅缺学号**

### 业务影响

| 功能 | 临时ID状态 | 需要真实学号 |
|------|-----------|-------------|
| 离校申请提交 | ✓ 可用（有楼栋数据，可路由） | - |
| 审批流程 | ✓ 可用 | - |
| 正式记录存档 | ✗ 不可用 | ✓ 必须 |
| 学工系统对接 | ✗ 不可用 | ✓ 必须 |
| 后续追溯查询 | ✗ 不可用 | ✓ 必须 |

**短期：** 这271名研究生可以正常使用离校申请系统  
**长期：** 必须补充真实学号，否则无法进行正式存档和系统对接

## 填写说明

### 1. 打开文件

使用Excel、WPS或其他表格软件打开 `missing_student_no_request.csv`

### 2. 填写学号

在"学号（待补充）"列中，根据学生的姓名、学院、专业等信息，填写正确的学号。

**注意事项：**
- 学号必须准确无误（用于官方记录）
- 如果某学生确实没有学号，请在"备注"列说明原因
- 保持其他列数据不变

### 3. 保存文件

填写完成后，保存为CSV格式（UTF-8编码），保持原文件名。

## 数据处理流程

### 当前状态

```
File2原始数据 → 学号字段为空 → 生成临时ID (TMP2026_XXXX) → File5
                                                         ↓
                                          可用于离校流程（短期）
                                          但无法正式存档（长期）
```

### 补充学号后

```
用户填写CSV → 更新File2 → 重新合并 → 更新File5 → 数据迁移
                                              ↓
                            临时ID替换为真实学号 → 可正式存档
```

### 数据更新脚本

提供数据后，系统将执行：

```bash
# 1. 更新File2中的学号字段
python3 backend/scripts/update_student_numbers.py \
  --filled-csv backend/data/missing_student_no_request.csv \
  --file2 backend/data/2026届预计毕业生5675人.csv \
  --output backend/data/2026届预计毕业生5675人_updated.csv

# 2. 重新合并生成File5
python3 backend/scripts/merge_student_data.py \
  --file1 backend/data/1-5830名毕业生（含研究生）.csv \
  --file2 backend/data/2026届预计毕业生5675人_updated.csv \
  --output backend/data/file5_students_merged_v2.csv

# 3. 数据库迁移（将临时ID对应的记录更新为真实学号）
python3 manage.py migrate_temp_ids \
  --mapping backend/data/missing_student_no_request.csv
```
