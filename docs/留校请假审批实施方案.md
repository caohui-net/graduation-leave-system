## 需求总结

1. **新增流程**：留校审批、请假审批
2. **入口改造**：登录后选择流程类型（离校/留校/请假）
3. **留校审批**：一级审批（辅导员），添加时间起止和原因选择
4. **请假审批**：前置展示，暂不开放业务
5. **代码复用**：最大化利用现有离校流程代码

def get_approval_flow(application_type):
    """根据申请类型返回审批流程"""
    if application_type == 'stay_school':
        return ['counselor']  # 留校：仅辅导员
    elif application_type == 'leave_school':
        return ['dorm_manager', 'counselor']  # 离校：宿管→辅导员
    elif application_type == 'leave_request':
        return []  # 请假：暂不开放
    return ['dorm_manager', 'counselor']

def create_approval_records(application):

### 3. API接口调整

**修改**: `backend/apps/applications/views.py`

```python
class ApplicationViewSet(viewsets.ModelViewSet):
    def create(self, request):
        data = request.data
        app_type = data.get('application_type', 'leave_school')
        
        # 留校申请验证
        if app_type == 'stay_school':
            required = ['stay_start_date', 'stay_end_date', 'stay_reason']
            if not all(data.get(field) for field in required):
                return Response({'error': '留校申请缺少必填字段'}, status=400)
        
        application = Application.objects.create(
            student=request.user,
            application_type=app_type,
            stay_start_date=data.get('stay_start_date'),
            stay_end_date=data.get('stay_end_date'),
            stay_reason=data.get('stay_reason'),
            # ...其他字段
        )
        
        create_approval_records(application)
        return Response(ApplicationSerializer(application).data)
```

## 三环境架构实施

### 环境定义

**dev（开发环境）**:
- 功能开发、联调、自测
- 可使用测试数据和模拟依赖

**staging（预发布环境）**:
- 与生产环境架构一致
- 用于验收、回归、压测

**prod（生产环境）**:
- 只部署经过验证的版本
- 所有变更可回滚、可观测

### 配置隔离

**1. 环境配置文件**

```
backend/config/
├── __init__.py
├── base.py          # 基础配置
├── dev.py           # 开发环境
├── staging.py       # 预发布环境
└── prod.py          # 生产环境
```

**2. 环境变量区分**

```bash
# .env.dev
DJANGO_SETTINGS_MODULE=config.dev
DATABASE_URL=postgresql://localhost/graduation_dev
ENABLE_STAY_SCHOOL=true

# .env.staging
DJANGO_SETTINGS_MODULE=config.staging
DATABASE_URL=postgresql://staging-db/graduation_staging
ENABLE_STAY_SCHOOL=false

# .env.prod
DJANGO_SETTINGS_MODULE=config.prod
DATABASE_URL=postgresql://prod-db/graduation_prod
ENABLE_STAY_SCHOOL=false
```

**3. Feature Flag控制**

```python
# backend/config/base.py
FEATURE_FLAGS = {
    'stay_school_approval': os.getenv('ENABLE_STAY_SCHOOL', 'false') == 'true',
    'leave_request_approval': os.getenv('ENABLE_LEAVE_REQUEST', 'false') == 'true',
}

# backend/apps/applications/views.py
from django.conf import settings

class ApplicationViewSet(viewsets.ModelViewSet):
    def create(self, request):
        app_type = request.data.get('application_type')
        
        # Feature flag检查
        if app_type == 'stay_school' and not settings.FEATURE_FLAGS['stay_school_approval']:
            return Response({'error': '功能未开放'}, status=403)
```

### 数据库迁移策略

**顺序**: dev → staging验证 → prod执行

```bash
# 1. dev环境：生成并执行迁移
cd backend
python manage.py makemigrations applications
python manage.py migrate --settings=config.dev

# 2. staging环境：验证迁移
python manage.py migrate --settings=config.staging --dry-run
python manage.py migrate --settings=config.staging

# 3. 准备回滚SQL（prod前）
python manage.py sqlmigrate applications 0001 > rollback.sql

# 4. prod环境：执行迁移（备份后）
# 先备份数据库
pg_dump graduation_prod > backup_$(date +%Y%m%d_%H%M%S).sql
# 执行迁移
python manage.py migrate --settings=config.prod
```

**向前兼容原则**:
- 先加字段，默认值确保旧代码兼容
- 不删除字段，只标记废弃
- 数据迁移分批执行

## 风险与注意事项

1. **数据迁移**: 生产环境迁移前备份数据库
2. **向后兼容**: application_type有默认值，不影响旧代码
3. **代码复用**: 尽量复用现有ApplicationSerializer、权限验证等
